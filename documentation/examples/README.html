Examples

The gallery mirrors the demo app (/dvl-demo) — each entry will become a page with the rendered figure, the complete spec, and a walkthrough of the one idea it adds. The demo presets, in teaching order:

Example Demonstrates
scatter marks, scales, axes
rect / text / ticks / arrows more marks over the same data
line / area aggregate marks, series via stroke: ordinal(...)
bars multi-output layouts: bandX + barY + .pick()
stacked stackY replacing barY
shared one scale object across layers
subplots / auto-row / auto-col the subplot model; panels as a virtual table
facet container + eq join
free-scales { shared: false } per-panel domains
sized-panels pixel-passthrough channels — panel boxes DERIVED in SQL from budgets (window functions), never hand-authored pixels
nesting .nest() multi-layer panels
fk-join via() cross-mark references
treemap treemap + fitText labels at computed-center vias (dual-substrate: runs in SQL when the data is there)
fit-text labels fitted INSIDE differently sized bars: barY's { zero: true } (no degenerate min bar), a via borrows each bar's solved box (anchor/baseline center the label inside it), and a bare fitText() — inputs inferred from the mark's channels (it sizes exactly what it draws), per-box + rotated + uncapped by default, the box bounds the solve
force-rects a node-link diagram of member rects (border color by community, white fill): edges attach to rect BOUNDARIES via route({ src, dst }).pick('x1','y1','x2','y2') (no manual center resolution — route derives the center from the box and clips to the edge), and each rect hosts a nested plot of its outgoing follows (nest('follows', { on: eq('src','id') }) — the FK itself), no axes
force force facets — the partition model: community panels with one simulation PER PANEL (perCell — the operator default in a template), and TWO edge layers over the same follows: a nested layer whose panel σ is DERIVED from the FK catalog (the cross-community follow is in neither panel's data), over a top-level { frame: srcVia } underlay (all edges, each assigned to its src's panel)
ER diagram schema introspection + nesting + composite-key via() (see branch exp/er-diagram-selfvis)

All presets build on five base tables — penguins (with island), members/follows (a 12-person social graph), readings (station temperatures), budgets — plus SQL-derived views (fromQuery): species, by_species, stacks, highlights, pairs, communities, and the sized-panels geometry. No hand-authored pixel tables remain.

Not yet demo presets, but worked in features/via.md: via resolutions — last-quote-before-order ({ fn: 'last', order: 'ts' }), weighted totals over a match set ({ spend: { fn: 'sum', arg: expr`f.price * f.qty` } }), opaque JS reductions (farthest-city radius), and cross-table borrows (db.table('penguins').via(...), arriving in data space).
