pow

The pow scale trains the same continuous {lo, hi} domain as linear but applies a power transform before normalizing: t = (valueᵏ − loᵏ) / (hiᵏ − loᵏ), mapped to [0, 1], where k is the exponent (default 2). It emphasizes the top of the range (k > 1) or the bottom (k < 1). sqrt is pow with k = ½; linear is pow with k = 1.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: pow("body_mass_g"),           // exponent defaults to 2
}).axis("x").axis("y");
return v;

Body mass under a squared transform — larger values are pushed apart, smaller ones compressed.

Options

  • Fitted — pow("col") fits its domain now.
  • Deferred / shared — const S = pow(); S("col") shares; reuse to union domains. S(4000) is a datum in the domain.

ScaleOpts (second argument):

option type default meaning
shared boolean solver decides pin (true) or forbid (false) domain sharing
side "top"|"right"|"bottom"|"left" auto pin the axis side
domain domain treatment "nice" "data", "nice", "zero", "unbounded", or { explicit: [lo, hi] }
params Record<string, Expr> {} carries scale parameters (see the exponent note below)
axis AxisOptions | false drawn axis config, or false to hide

Exponent (k). The transform reads exponent from the binding options; it defaults to 2. Two exponents are exact today: k = 1 (identity, same as linear) and k = 2 (x·x). Other exponents fall back to a placeholder approximation (√(x²) ≈ |x|) — the source has no pow leaf in its scalar algebra, so k ∉ {1, 2} is not yet faithful. For the common k = ½ case use the dedicated sqrt scale, which is exact. Ticks use nice-linear.

On a colour channel pow is a sequential ramp (default blues); its legend is a gradient bar.

Examples

Anchor the domain at zero — a squared scale is usually read against a zero baseline.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: pow("body_mass_g", { domain: "zero" }),
}).axis("x").axis("y");
return v;

Explicit domain so the emphasis is stable across renders.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: pow("body_mass_g", { domain: { explicit: [2500, 6500] } }),
}).axis("x").axis("y");
return v;

Shared squared domain across two tables.

const heavy = db.table("penguins").top(1, { col: "body_mass_g", desc: true }, { partition: ["species"] });
const v = canvas(640, 440);
const x = linear("bill_length_mm");
const y = pow("body_mass_g");
v.dot("penguins", { x, y, fill: "#cfd3dc" });
v.dot(heavy,      { x, y, fill: "#e15759" });
return v;

pow colour ramp — a squared emphasis on the fill channel.

const v = canvas(680, 460);
const p = v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: pow("body_mass_g"),
  r: 5,
});
p.legend("fill", { placement: "right" });
return v;
