threshold

The threshold scale is binned with author-specified breakpoints. Unlike every other numeric scale it has no train: its domain is static — you give it the cut points, and each value maps to the bin whose lower bound it exceeds (D3's scaleThreshold). N breakpoints define N+1 output bins. Because nothing is fitted from data, no domain fit ever materializes — the domain is ordinal, ordered, and non-interpolable.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: threshold("body_mass_g", { params: { breaks: [3500, 4500, 5500] } }),
}).axis("x").axis("y");
return v;

Mass split at 3500 / 4500 / 5500 into four classes, each a distinct colour — a classed encoding with boundaries you choose rather than equal-width bins.

Options

  • Fitted — threshold("col") binds the column; the breakpoints (not the data) define the bins.
  • Deferred — const S = threshold(); S("col") is callable per channel. There is no domain to union — the breakpoints are the whole story.

ScaleOpts (second argument):

option type default meaning
shared boolean solver decides facet-level sharing hint (no data domain to share)
side "top"|"right"|"bottom"|"left" auto pin the axis side (positional use)
domain domain treatment — no data-derived domain; the breakpoints are the domain
params Record<string, Expr> {} carries breaks (the breakpoint array) and the output range/palette
axis AxisOptions | false drawn axis config, or false to hide

Breakpoints (breaks). The cut points come from the binding's breaks option — an ascending array of numbers. N breaks → N+1 bins; each bin maps to one output entry (a categorical palette entry, or a colour-ramp step on a colour channel). With no breaks, everything maps to the first output entry. The domain is ordinal/ordered, so the legend is a set of discrete blocks, not a gradient.

Implementation status. The scalar algebra has no CASE/>= boolean yet, so the current threshold apply is a placeholder: only evenly spaced breakpoints are expressed exactly (via floor((value − breaks[0]) / step)), and in the present source the bucketing collapses to the first output entry pending that algebra extension. Arbitrary (unevenly spaced) breaks and the full cascade await a CASE-expression primitive. Prefer quantize (equal-width bins, which is wired) when you want a data-driven classing today; reach for threshold when you need to state the exact boundaries and can accept the current limitation.

On a colour channel each bin maps to one palette entry (default blues, sampled) — a classed colour encoding with your chosen boundaries.

Examples

Author-chosen mass classes on the colour channel, with a discrete legend.

Illustrative, not yet wired. threshold's domain is static (author breakpoints, no train), so it has no fitted domain rows for a swatch legend to enumerate — the discrete-legend case below does not render end-to-end today. It is shown for the intended shape only. The apply is also a placeholder that currently expresses only evenly-spaced breaks (see the source note in components/scales/threshold.ts); for a wired, data-fitted binned colour legend use quantize.

// NOT RUNNABLE (see note above) — the intended discrete-legend shape:
const v = canvas(680, 460);
const p = v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: threshold("body_mass_g", { params: { breaks: [3500, 4500, 5500] } }),
  r: 5,
});
p.legend("fill", { placement: "right" });   // discrete class blocks (not yet wired)
return v;

Evenly spaced breakpoints — the case the current transform expresses exactly.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: threshold("body_mass_g", { params: { breaks: [3000, 4000, 5000, 6000] } }),
}).axis("x").axis("y");
return v;
