color

Colour is an OUTPUT SPACE, not a scale family. The same scale emits a number on a positional channel and a colour on a colour channel (fill/stroke) — the choice is made by the channel, not the scale name. So:

  • color("col") is a continuous colour scale: a linear with a sequential-scheme default. Its apply normalizes t = (v − lo) / (hi − lo) ∈ [0,1] and, on a colour channel, indexes a 256-step ramp — a smooth gradient, not banded. linear("col") (or any continuous scale) on a colour channel behaves identically.
  • ordinal("col") on a colour channel is a categorical palette — distinct hues indexed by category rank (see ordinal).

A scheme names the palette. It resolves once to an array: a categorical scheme stays a fixed hue list; a sequential/diverging scheme is pre-sampled to 256 steps so the continuous apply is index(ramp, floor(t·255)) (dual-substrate — the same index lowers to both SQL and JS; no colour-interpolation primitive).

const v = canvas(680, 460);
const p = v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: color("body_mass_g"),   // heavier birds → darker; a smooth blues ramp
  r: 5,
});
p.legend("fill", { placement: "right" });   // continuous colour → a GRADIENT legend
return v;

A scatter where fill encodes mass along a smooth blues ramp, with a gradient legend bar in the right gutter.

Options

color(column, opts?) — a continuous colour scale (default scheme blues). Because colour is an output space, the colour behaviour also comes from ordinal(column) (categorical) and from any continuous scale on a colour channel.

Option Type Default Meaning
scheme scheme name "blues" (continuous) / "tableau10" (ordinal) named palette; resolved once — sampled to 256 steps for a ramp, kept as-is for categorical
range array — explicit palette override (wins over the scheme)
shared boolean — reuse the object to declare "same quantity" (one unioned domain, one legend)
domain domain treatment "data" domain treatment; a continuous colour ramp is not niceable (the ramp is a fixed palette)

The continuous domain is a 1-row {lo, hi} fit (min/max of the column), cross-joined to every mark, envelope-unionable. The apply is the same normalized t as linear; scaledOut emits a number for a positional channel and a colour (index into the resolved ramp) for style⟨color⟩. A colour-declared output that computes a non-colour is a build error — the old "fill: linear yields a raw number" type-lie is gone.

Schemes

From src/lib/dvl/components/schemes.ts:

Family Channel result Default Names
categorical (ordinal) discrete palette tableau10 tableau10, observable10, category10
sequential (color, or any continuous scale) 256-step smooth ramp blues blues, viridis, turbo
diverging 256-step ramp (middle stop = pivot) — rdbu, blueorange

Categorical palettes are fixed hue arrays indexed by rank (mod length). Ramps are given as a handful of control stops and piecewise-linearly interpolated to 256 colours (sampleRamp). The defensive fallbacks live in scales/_shared.ts (CATEGORICAL_PALETTE, SEQUENTIAL_FALLBACK).

Autolegend

A colour channel's legend is decided by whether its scale is continuous or categorical:

  • continuous (color, linear, …) → a gradient ramp bar (thin colour bands sampling the resolved ramp) with axis-style min/mid/max labels.
  • categorical (ordinal) → a swatch set — one swatch + label per category, using the same resolved palette as the marks.

Ask for it with .legend(channel, { placement }). There is no bespoke legend renderer: the legend is lowered to an ordinary plot over the scale's domain, drawn by the same marks/layout machinery. placement picks the side: left | right | top | bottom | inside (inside is an in-plot overlay that reserves no gutter; position is an accepted alias for placement).

Examples

Continuous colour ramp on fill:

const v = canvas(680, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: color("body_mass_g"),
  r: 5,
});
return v;

Continuous colour with a gradient legend:

const v = canvas(680, 460);
const p = v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: color("body_mass_g"),
});
p.legend("fill", { placement: "right" });
return v;

Any continuous scale on a colour channel ramps the same way — linear on fill is a colour, not a number:

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: linear("body_mass_g"),   // identical to color("body_mass_g")
});
return v;

Pick a different sequential scheme:

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: color("body_mass_g", { scheme: "viridis" }),
});
return v;

Categorical colour + a swatch legend (the ordinal case):

const v = canvas(680, 460);
const p = v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("body_mass_g"),
  fill: ordinal("species"),
});
p.legend("fill", { placement: "right" });   // 3 swatches + labels
return v;

Legend as an in-plot overlay (no gutter reserved):

const v = canvas(640, 460);
const p = v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: ordinal("species"),
});
p.legend("fill", { placement: "inside" });
return v;

Top-docked colour legend:

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("body_mass_g"),
  fill: ordinal("species"),
}).legend("fill", { placement: "top" });
return v;

Sharing one continuous colour scale across layers — reuse the object so both layers union one domain and share one legend:

const v = canvas(680, 460);
const heat = color("body_mass_g");          // one object → "same quantity"
const p = v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: heat,                                // reuse the SAME object across marks
  r: 5,
});
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: heat,                                // shares one domain + one legend
  r: 2,
});
p.legend("fill", { placement: "right" });
return v;
