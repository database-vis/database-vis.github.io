linear

The linear scale is an operator that trains a continuous domain from your data — the {lo, hi} min/max envelope of a column — and applies a per-row normalization to the relative interval [0, 1]: t = (value − lo) / (hi − lo). It emits nothing pixel-shaped itself; the frame affine maps that normalized t onto the axis box. It is the identity transform of the continuous family — the default numeric scale (Vega-Lite/Plot linear).

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: ordinal("species"),
}).axis("x").axis("y");
return v;

Bill length against flipper length. Each linear reduces its column to a {lo, hi} domain, then normalizes every row into [0, 1]; the frame affine places those onto the pixel box.

Options

Two authoring forms:

  • Fitted — linear("col") fits its domain to col now. Table-scoped identity: two marks over one table both binding linear("mass") share a scale (domains union); the same column name on a different table is a different quantity.
  • Deferred / shared — const S = linear(); S("col") is deferred. Reuse the same object across marks to declare "same quantity" — the compiler unions the domains (a gather-scatter), so the layers share one frame and one axis. S(4000) is a datum — a literal placed in S's domain (and it trains the domain like any member).

Second argument is a ScaleOpts bag:

option type default meaning
shared boolean solver decides true pins domain sharing across facets/panels; false trains a domain per panel (each draws its own axis)
side "top"|"right"|"bottom"|"left" auto pin the axis to a side (else assignSides places it)
domain domain treatment "nice" "data" (raw min/max), "nice" (round to nice bounds), "zero" (extend to include 0), "unbounded", or { explicit: [lo, hi] }
params Record<string, Expr> {} scale-specific parameters (none for linear)
axis AxisOptions | false drawn axis config, or false to hide the axis

AxisOptions: { ticks?: number | { count?, algorithm? }, side?, tickLength? (5), fontSize? (10) }.

Domain treatments are recorded API intent; nice is the derived default because a linear domain is niceable. Ticks use the nice-linear algorithm.

On a colour channel any continuous scale is a sequential ramp: bind linear("col") to fill/stroke and its t indexes a 256-step colour ramp (default scheme blues) — smooth, not banded. The legend for a continuous colour is a gradient bar, not swatches. (The explicit-intent factory for this is color; linear on a colour channel is identical.)

Examples

Explicit domain — force the y-axis to start at zero regardless of the data minimum.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("body_mass_g", { domain: "zero" }),
}).axis("x").axis("y");
return v;

A fixed explicit domain, so two renders are comparable even if their data differ.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm", { domain: { explicit: [30, 60] } }),
  y: linear("body_mass_g",    { domain: { explicit: [2500, 6500] } }),
}).axis("x").axis("y");
return v;

Shared scale across two tables — reuse the same objects so domains union and the layers sit in one comparable frame.

const heavy = db.table("penguins").top(1, { col: "body_mass_g", desc: true }, { partition: ["species"] });
const v = canvas(640, 440);
const x = linear("bill_length_mm");
const y = linear("body_mass_g");
v.dot("penguins", { x, y, fill: "#cfd3dc" });   // the full cloud
v.dot(heavy,      { x, y, fill: "#e15759" });    // heaviest bird per species
return v;

Continuous colour ramp — bind linear to fill; the legend is a gradient.

const v = canvas(680, 460);
const p = v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: linear("body_mass_g"),   // heavier → darker; a smooth blues ramp
  r: 5,
});
p.legend("fill", { placement: "right" });
return v;

Free (per-panel) scales — shared: false trains a domain per panel, so each panel's points fill its own frame and each draws its own axis. Panels are data-driven: one host row → one panel.

const species = db.table("penguins").groupby(["species"], {});   // one row per species
const v = canvas(760, 260);
v.rect(species, { ...eqX("species").pick("x", "w"), y: 12, h: 236, fill: "#f5f6fa" })
 .dot("penguins",
   { x: linear("bill_length_mm", { shared: false }),
     y: linear("body_mass_g",    { shared: false }), fill: "#59a14f" },
   { on: eq("species", "species") });
return v;

Datum in the domain — a deferred scale lets a constant reference line share the frame's units. y(4000) is a literal placed through y, so the rule sits at mass 4000 in the same frame.

const v = canvas(640, 460);
const y = linear();  // deferred, so y(4000) is a datum placed through the same frame
v.dot("penguins", { x: linear("bill_length_mm"), y: y("body_mass_g") }).axis("x").axis("y");
v.rule("penguins", { x: 0, x2: 640, y: y(4000) });   // horizontal line at mass = 4000
return v;

Hide the axis on a secondary channel with axis: false.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("body_mass_g", { axis: false }),
});
return v;
