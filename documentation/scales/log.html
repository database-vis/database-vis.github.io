log

The log scale trains the same continuous {lo, hi} domain as linear (a min/max reduction) but interpolates in log space: t = (ln value − ln lo) / (ln hi − ln lo), mapped to [0, 1]. The frame affine then places t on the axis. Use it when a quantity spans orders of magnitude and you want ratios to read as equal distances. It assumes a positive domain — the ln of a non-positive value is NULL, so that mark drops (as any out-of-domain point does).

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: log("body_mass_g"),
}).axis("x").axis("y");
return v;

Body mass on a log y-axis: the same data, but ratios (not differences) become the visual metric.

Options

Same two forms as every continuous scale:

  • Fitted — log("col") fits its domain to col now (table-scoped identity).
  • Deferred / shared — const S = log(); S("col") shares the object; reuse it across marks to union domains into one comparable frame. S(1000) is a datum in the domain.

ScaleOpts (second argument):

option type default meaning
shared boolean solver decides pin (true) or forbid (false) domain sharing across panels
side "top"|"right"|"bottom"|"left" auto pin the axis side
domain domain treatment "nice" "data", "nice", "zero", "unbounded", or { explicit: [lo, hi] }
params Record<string, Expr> {} no log-base parameter is exposed — the transform is natural log (ln), and the base cancels in the ratio so the choice of base does not affect t
axis AxisOptions | false drawn axis config, or false to hide

Ticks use nice-linear (linear-spaced ticks in log space). A log-nice 1,2,5·10ᵏ scheme is not wired yet. Keep the domain strictly positive — a zero treatment or an explicit domain that includes 0 or negatives yields NULL at the boundary.

On a colour channel log is a sequential ramp (default scheme blues), same as any continuous scale; its legend is a gradient bar.

Examples

An explicit positive domain, so the axis is stable across renders.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: log("body_mass_g", { domain: { explicit: [2500, 6500] } }),
}).axis("x").axis("y");
return v;

Two y quantities on one plot — a linear x and a log y are independent scales and each gets its own axis.

const v = canvas(640, 460);
v.line("penguins", {
  x: linear("year"),
  y: log("body_mass_g"),
}).axis("x").axis("y");
return v;

Shared log domain across two tables — reuse the object so both layers share one log frame.

const heavy = db.table("penguins").top(1, { col: "body_mass_g", desc: true }, { partition: ["species"] });
const v = canvas(640, 440);
const x = linear("bill_length_mm");
const y = log("body_mass_g");
v.dot("penguins", { x, y, fill: "#cfd3dc" });
v.dot(heavy,      { x, y, fill: "#e15759" });
return v;

Log colour ramp — a heavy-tailed quantity reads better on a log fill.

const v = canvas(680, 460);
const p = v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: log("body_mass_g"),
  r: 5,
});
p.legend("fill", { placement: "right" });
return v;
