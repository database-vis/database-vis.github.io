point

point maps a categorical column to a single evenly-spaced position — D3/VL scalePoint, like a band with zero bandwidth. Each category sits at its band center: pos(i) = (idx + 0.5)/n, so a categorical dot/line/tick plot centers every category in its slot. It trains the keyed category table (value, idx, n) and applies a keyed join, so both the data points and the axis ticks land at the same position (no anchor split). Single output — a position, no width — so it binds like linear: x: point("species"), no .pick().

const v = canvas(640, 460);
v.dot("penguins", {
  x: point("species"),          // 3 categories → x at 1/6, 3/6, 5/6 of the frame
  y: linear("body_mass_g"),
}).axis("x").axis("y");
return v;

Body mass by species, each species centered in its third of the x-axis.

Options

point(column, opts?). The column is the categorical field whose distinct values form the domain.

Option Type Default Meaning
shared boolean — reuse the object across bindings to declare "same categories" (one unioned domain, one axis)
side "top" | "right" | "bottom" | "left" — pin the axis side
domain domain treatment "data" domain treatment (recorded intent)
axis AxisOptions | false — tick/label config; each category labels its own position

The domain is categorical with ordered: true — categories carry positional meaning (order along the axis). Train is the keyed (value, idx, n) table; apply computes pos = (idx + 0.5) / n. Ticks are ordinal (one per category), placed at the same centers as the data.

Examples

Point scale on x, continuous on y — a category-vs-quantity scatter:

const v = canvas(640, 460);
v.dot("penguins", {
  x: point("species"),
  y: linear("bill_length_mm"),
}).axis("x").axis("y");
return v;

Add a categorical colour so the point position and the fill agree:

const v = canvas(640, 460);
v.dot("penguins", {
  x: point("species"),
  y: linear("body_mass_g"),
  fill: ordinal("species"),
}).axis("x").axis("y");
return v;

Point on y for a horizontal categorical layout:

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: point("island"),          // Torgersen / Biscoe / Dream centered on y
}).axis("x").axis("y");
return v;

Ticks as a categorical mark — tick at each category center:

const v = canvas(640, 460);
v.tick("penguins", {
  x: point("species"),
  y: linear("flipper_length_mm"),
}).axis("x").axis("y");
return v;

point vs bandX: point gives one centered coordinate (no width — for dots/lines/ticks), whereas bandX gives an (x, w) pair (for bars/rects that fill their slot). Use point when the mark is a glyph at a position, bandX when the mark spans the category's width.

See also

  • ordinal — category → a discrete range (palette or slot), indexed by rank.
  • bandX — category → positional band with width (bars, rects).
  • color — colour as an output space.
