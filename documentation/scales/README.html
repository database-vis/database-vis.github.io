Scales

Scales are operators that train a domain from your data (in SQL, in the database) and apply a per-row normalization. Bind one to a channel — x: linear("bill_length_mm") — or share one across layers by reusing the object. Options ride the second argument: linear("col", { shared: false, domain: "nice", axis: { ticks: 4 } }), and on a color channel a scheme names the palette: linear("mass", { scheme: "viridis" }).

Continuous (train min/max)

Scale Apply Notes
linear (v − lo) / (hi − lo) the workhorse
log interpolates in ln-space positive domains
sqrt square-root transform
pow power transform exponent option (default 2)
symlog bi-symmetric log handles zero and negatives
color position on a color ramp continuous fill — a linear with a sequential-scheme default; any continuous scale on a color channel behaves the same (see Color)

Categorical (train distinct)

Scale Apply Notes
ordinal category → range[index] palette for fill, or any discrete range
point category → centered position band with zero width

Binned

Scale Apply Notes
quantize N equal-width bins over [lo, hi] trained
threshold authored breakpoints → bins no training (a static domain)

Color

Color is an output space, not a scale family. The same scale emits a number on a positional channel and a color on a color channel (fill/stroke) — decided by the channel, not the scale name. So fill: linear("mass") is a sequential ramp (not a stray number), fill: ordinal("species") is a categorical palette, and color("mass") is just linear with a sequential default. The palette comes from a named scheme (resolved once to a 256-step ramp for continuous, a fixed array for categorical); override with { scheme } or an explicit { range: [...] }.

Family Channel → Default scheme Schemes
continuous (linear/log/sqrt/pow/symlog/color) sequential ramp blues blues, viridis, turbo, diverging rdbu/blueorange
categorical (ordinal) discrete palette tableau10 tableau10, observable10, category10
binned (quantize/threshold) discrete bins (sequential) any sequential, sampled to the bin count

Sharing. Reusing one scale object across bindings declares "same quantity" — one unioned domain, one axis. Facet panels share domains by default; { shared: false } gives each panel its own. Choosing fill: a categorical column → ordinal; a numeric intensity → color (or any continuous scale — they all ramp on a color channel). A string column under color()/linear() still fails at train time (min/max over text) — that is a data-type error, not the old type-lie (a continuous scale on fill now always yields a color, never a raw number).

Adding your own scale

A scale is an operator — the same op({ … }) factory layouts use — with a trained domain and an apply that normalizes each row against the fit.

import { op } from "$lib/dvl/operator/operator";
import { continuous } from "$lib/dvl/components/scales/types";
import { scaledOut } from "$lib/dvl/components/scales/_shared";
import { inputCol } from "$lib/dvl/algebra/impl";

const clampLinear = op({
  name: "clampLinear",
  domain: continuous((b) => ({ kind: "reduce", aggs: {
    lo: { fn: "min", col: inputCol(b, "value") }, hi: { fn: "max", col: inputCol(b, "value") } } })),
  apply: (b) => scaledOut(b, /* clamp((v − lo) / (hi − lo), 0, 1) — a ScalarExpr */ …),
  ticks: "nice-linear",   // earns a labelled axis
  scheme: "blues",        // default palette when bound to a colour channel
});
  • domain — continuous(trainFn) (a 1-row {lo, hi} fit) or categorical(trainFn) (the keyed value table). The train runs in SQL, in the database.
  • apply(b) returns a RelOp computing the normalized t; wrapping it in scaledOut(b, t) makes the SAME operator emit a number on a positional channel and a colour on a colour channel — the binding-polymorphic apply (Color).
  • ticks names the tick algo for the axis; scheme the default palette.

Register it and expose a factory — exactly like a layout:

registries.operators.register(clampLinear.name, clampLinear);
export const clampLinear = scaleFactory("clampLinear");   // → clampLinear("mass")

Marks (defineMark), scales, and layouts (op) are the three extension points; all three register into the Registries and surface through a builder factory.
