connect

connect is the PAIR sibling of the sequence family. Where path/line/area AGGREGATE N points into one shape, connect keeps them 1:1 and draws a segment from each point to its SUCCESSOR in order — a LEAD window per group (SQL LEAD(...) OVER (PARTITION BY … ORDER BY …)). N points yield N−1 visible segments per group: the last point of each group has no successor, so its row draws a degenerate (zero-length, invisible) segment. Points sequence by the order channel, which defaults to x. A stroke colour partitions into groups AND colours the segments. Because each segment is its own mark, it is individually styleable — the key difference from path, whose whole polyline is one <path>.

// Each point → the next point in `order` (year), within a species — a LEAD window per group.
// Same inline-modelled time series as the trajectory example; each segment is its own mark.
const series = db.table("penguins").groupby(["species", "year"],
  { bill: avg("bill_length_mm"), mass: avg("body_mass_g") });
const v = canvas(680, 460);
v.connect(series, {
  x: linear("bill"),
  y: linear("mass"),
  order: field("year"),          // each year → the next year, within a species
  stroke: ordinal("species"),
  strokeWidth: 2,
}).axis("x").axis("y");
return v;

Per species, one segment between each pair of consecutive yearly means (N−1 segments per group).

Options

connect is a stroked pair mark. It has the same channel surface as path — no baseline — but its cardinality is pairs, not aggregate: each input row becomes one segment to its successor. The stroke channel both partitions the groups (the LEAD window's PARTITION BY) and colours the segments.

Channel Type Default Meaning
x positional-x — point x (the segment's start; its end is the successor's x)
y positional-y — point y
order number x the SEQUENCE column — successor = next row by this within the group
stroke color — segment colour; distinct values partition the LEAD window into groups
strokeWidth pixel 1 segment thickness (px)
strokeDasharray text — dash pattern, e.g. "4 2"
strokeOpacity number 1 segment alpha
opacity number 1 whole-mark alpha
title text — per-segment <title> hover tooltip

order is an EXPRESSION channel — field("t") (aka col("t")) names a raw column; a bare string is a literal. Group tails draw an invisible zero-length segment (start == end), so a group of N points shows N−1 lines.

Examples

Basic connect — a single group, sequenced by x (default order), chaining consecutive points. A connect segment is stroke-only, so it needs a stroke to be visible (an unstroked <line> paints nothing):

const yearly = db.table("penguins").groupby(["year"], { mass: avg("body_mass_g") });
const v = canvas(480, 320);
v.connect(yearly, { x: linear("year"), y: linear("mass"), stroke: "#4e79a7", strokeWidth: 2 })
 .axis("x").axis("y");
return v;

Multi-group by colour, with a legend — one chain of segments per species:

const series = db.table("penguins").groupby(["species", "year"],
  { bill: avg("bill_length_mm"), mass: avg("body_mass_g") });
const v = canvas(680, 460);
const p = v.connect(series, {
  x: linear("bill"), y: linear("mass"),
  order: field("year"), stroke: ordinal("species"), strokeWidth: 2,
});
p.axis("x").axis("y").legend("stroke", { placement: "right" });
return v;

Per-segment styling — the payoff of connect over path: each segment is its own mark, so a per-segment column drives its width or dash. Here segment thickness rides a data attribute:

// a track where each hop carries its own weight (raw-SQL escape hatch)
const hops = db.derive("hops", `
  SELECT * FROM (VALUES
    ('a',1,0,0,1),('a',2,10,5,4),('a',3,5,10,2),
    ('b',1,0,10,3),('b',2,10,0,1))
  AS t(trip, step, lon, lat, w)`);
const v = canvas(320, 320);
v.connect(hops, {
  x: linear("lon"), y: linear("lat"),
  order: field("step"),
  stroke: ordinal("trip"),
  strokeWidth: px("w"),          // each segment's width from its own row (pixel column)
}).axis("x").axis("y");
return v;

Time-ordered, non-monotone — the LEAD window sorts within each group by order, so the chain follows time even when it doubles back on x:

const track = db.derive("track", `
  SELECT * FROM (VALUES
    ('a',1,0,0),('a',2,10,5),('a',3,5,10),
    ('b',1,0,10),('b',2,10,0))
  AS t(trip, step, lon, lat)`);
const v = canvas(240, 240);
v.connect(track, {
  x: linear("lon"), y: linear("lat"),
  order: field("step"), stroke: ordinal("trip"),
}).axis("x").axis("y");
return v;

Trip a (3 points) draws 2 segments, trip b (2 points) draws 1; each trip's tail is invisible.
