dot

The dot mark emits one <circle> per row. x and y place its centre through positional scales; r is its pixel radius; fill/stroke paint it. It is the point mark (Vega-Lite point/circle/square, Plot dot) — the workhorse for scatterplots and, layered, for anything that sits on a placed point.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("flipper_length_mm"),
  fill: ordinal("species"),
});
return v;

Bill length versus flipper length, one circle per bird, coloured by species. The linear scales train their domains from the data; ordinal maps each species to a categorical palette entry.

Options

x/y are the only required channels. Everything else has a draw-time default (radius 3, black fill). A bare string on any channel is a literal, not a column — fill: "species" paints every dot the CSS colour species; use field("species") (from $lib/dvl/ir/expr) or a scale binding to read a column.

channel kind default meaning
x positional-x — centre x (scale binding, layout output, or pixel literal)
y positional-y — centre y
r pixel 3 radius in pixels (bind a scale for a bubble chart)
shape shape circle glyph shape
fill color black interior paint
stroke color none outline paint
strokeWidth pixel — outline width
strokeDasharray text — SVG dash pattern
opacity number 1 whole-mark opacity
fillOpacity number 1 interior opacity
strokeOpacity number 1 outline opacity
title text — per-mark hover tooltip (SVG <title>); also a join carrier for via

r and strokeWidth are pixel channels scaled on both axes. A dot is centered on (x, y) by nature — it has no anchor option (unlike the box marks rect/image).

Examples

Colour and size encoded — a bubble chart. r is a raw pixel radius, so drive it with a channel expression: sqrt(body_mass_g) makes the area (∝ r²) proportional to mass, and / 10 scales the bubbles to a legible range. (A size scale — sqrt("body_mass_g") — is the wrong tool here: scales emit a relative [0, 1], which the frame affine turns into pixels only for positional channels; on a bare radius that [0, 1] is sub-pixel, so the dots vanish.)

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("bill_depth_mm"),
  r: cexpr`sqrt(body_mass_g) / 10`,   // area ∝ mass; the /10 keeps radii in a legible px range
  fill: ordinal("species"),
  fillOpacity: 0.7,
});
return v;

Two layers over one table, sharing the same x/y scale objects, so the label lands on its dot rather than re-deriving a position. field reads a column; dx/dy nudge the label a few pixels.

const v = canvas(640, 460);
const x = linear("bill_length_mm"), y = linear("body_mass_g");
v.dot("penguins", { x, y, fill: "#cfd3dc" });
v.text("penguins", {
  x, y,   // same scale objects → the label is placed relative to its dot
  text: field("species"), dx: 6, dy: -6, anchor: "start", fontSize: 10, fill: "#333",
});
return v;

Shared scales across two tables. Reusing the same x/y objects across marks over different tables declares them the same quantity: the compiler unions the domains, so both layers share one frame and compare directly. The highlights (heaviest bird per species) are modelled inline.

const highlights = db.table("penguins").top(1, { col: "body_mass_g", desc: true }, { partition: ["species"] });
const v = canvas(640, 440);
const x = linear("bill_length_mm");
const y = linear("body_mass_g");
v.dot("penguins",  { x, y, fill: "#cfd3dc" });  // the full cloud
v.dot(highlights,  { x, y, fill: "#e15759" });  // the heaviest bird per species
return v;

Axes. Ask a plot for its axes; each is lowered to an ordinary tick+text plot over the scale.

const v = canvas(640, 460);
v.dot("penguins", {
  x: linear("bill_length_mm"),
  y: linear("body_mass_g"),
  fill: ordinal("species"),
}).axis("x").axis("y").legend("fill");
return v;

Faceted (small multiples). A container rect per species hosts a nested dot plot keyed on species. The panel count comes from the data; shared scales make every panel's range identical.

const species = db.table("penguins").groupby(["species"], {});  // one row per species
const v = canvas(760, 260);
v.rect(species, { ...eqX("species").pick("x", "w"), y: px(12), h: 236, fill: "#f5f6fa" })
 .dot("penguins",
   { x: linear("bill_length_mm"), y: linear("body_mass_g"), fill: "#4e79a7" },
   { on: eq("species", "species") });   // child.species = host.species
return v;

Parallel coordinates. One normalized dot plot per attribute at a fixed x; title carries a stable tuple key so link.via can reconstruct each tuple's polyline across adjacent axes.

const p = db.table("penguins").withKey("id");
const attrs = ["bill_length_mm", "bill_depth_mm", "flipper_length_mm", "body_mass_g"];
const xs = [90, 300, 500, 670];
const v = canvas(740, 460);
const axes = attrs.map((a, i) => v.dot(p, {
  x: px(xs[i]), y: linear(a, { axis: false }),
  r: 3, fill: ordinal("species"), title: field("id"),
}));
for (let i = 0; i < attrs.length - 1; i++) {
  v.link(p, {
    ...axes[i].via(eq("id", "id")).pick({ x1: "x", y1: "y" }),
    ...axes[i + 1].via(eq("id", "id")).pick({ x2: "x", y2: "y" }),
    stroke: ordinal("species"), strokeWidth: 1, strokeOpacity: 0.3,
  });
}
return v;
