text

The text mark draws a string at each row's (x, y). It is the mark where the bare-string-is-a-literal rule bites hardest: text: "species" writes the literal word "species" on every point, while text: field("species") reads the species column and writes each row's value. (field is col from $lib/dvl/ir/expr — the EXPRESSION column, not the builder layout barY.) Because a label almost always sits on top of an existing point or box, text is typically a second layer that reuses the first layer's x/y scales, then nudges itself off the anchor with dx/dy.

const v = canvas(640, 460);
const x = linear("bill_length_mm"), y = linear("body_mass_g");
v.dot("penguins",  { x, y, fill: "#cfd3dc" });
v.text("penguins", {
  x, y,                                  // SAME scale objects → label lands on its dot
  text: field("species"),                // reads the species COLUMN
  dx: 6, dy: -6, anchor: "start", fontSize: 10, fill: "#333",
});
return v;

A grey dot cloud with each point labelled by its species, the text nudged 6px up-and-right of the dot so it does not sit under the marker.

Options

Positional x/y place the anchor; dx/dy are pixel nudges the draw applies after placement (so an axis label can sit a few px off its baseline). text is the content channel: bind it to field("col") for column values, or pass a bare string for a constant label. anchor and baseline set the SVG text-anchor / dominant-baseline; rotate spins the glyph in place. Multiline content arrives via the lines output of fitText (below) — you rarely set it by hand.

channel type default meaning
x positional-x — anchor x (scale binding, layout output, or pixel literal)
y positional-y — anchor y
dx pixel 0 horizontal nudge applied after placement (+ = right)
dy pixel 0 vertical nudge (− = up, + = down)
text text "" the string. field("col") = column; bare string = literal on every row
anchor shape middle* text-anchor: start | middle | end
baseline shape — dominant-baseline: top | middle | bottom (alphabetic if unset)
fontSize fontSize 10* glyph size in px (terminal pen param — never affine-placed)
rotate rotate 0 rotation in degrees
fill color — glyph colour
opacity number 1 glyph opacity

* pen defaults; anchor/fontSize/fill inherit the renderer's font style when unset.

The mark also reads a lines field (a JSON array of strings) when present: more than one line triggers the multiline draw. fitText produces lines; you normally get multiline text by composing with that layout rather than writing the channel directly.

Examples

A bare label layer — one constant word tagging a region of the plot. text: "Adelie" is a LITERAL, so it is written once at the given pixel point, not per row.

const v = canvas(640, 460);
const x = linear("bill_length_mm"), y = linear("body_mass_g");
v.dot("penguins", { x, y, r: 3, fill: "#cfd3dc" });
v.text("penguins", { x: px(200), y: px(40), text: "penguin body mass", fontSize: 14, fill: "#333" });
return v;

Column labels nudged off their points. dx/dy push the text to the dot's upper-right; anchor: "start" grows it rightward so it never overlaps the marker.

const v = canvas(640, 460);
const x = linear("bill_length_mm"), y = linear("body_mass_g");
v.dot("penguins",  { x, y, fill: ordinal("species") }).axis("x").axis("y");
v.text("penguins", {
  x, y, text: field("island"),
  dx: 5, dy: -4, anchor: "start", fontSize: 9, fill: "#555",
});
return v;

Rotated tick-style labels. rotate spins each glyph; combine with anchor: "end" for angled category labels that read up toward their point.

const v = canvas(640, 460);
const x = linear("bill_length_mm"), y = linear("body_mass_g");
v.dot("penguins",  { x, y, r: 3, fill: "#cfd3dc" });
v.text("penguins", {
  x, y, text: field("species"),
  rotate: -35, anchor: "end", fontSize: 9, fill: "#333",
});
return v;

Centred labels inside a box. anchor: "middle" + baseline: "middle" centre the text on a computed box midpoint — the position pattern used by every fitted-label example below.

const v = canvas(640, 460);
const bars = v.bar("penguins", {
  ...bandX("species", { paddingInner: 0.15 }).pick("x", "w"),
  ...barY("body_mass_g").pick("y", "h"),
  fill: ordinal("species"),
});
v.text("penguins", {
  ...bars.via(eq("species", "species")).pick({ x: "x", y: "y", w: "w", h: "h" }),
  text: field("species"), anchor: "middle", baseline: "middle", fill: "#fff", fontSize: 11,
});
return v;

Fit text in a box. text composes with the fitText refine layout, which sizes the string to its own w/h box. Inputs (text, w, h) are inferred from the mark's channels; .pick( "fontSize", "rotate") binds the solved size and per-box rotation. Long names in tall bars rotate to fit, short ones stay horizontal; the box itself bounds the (uncapped) solve.

const revenue = db.table("orders").rollup(
  db.table("products"), { left: "product_id", right: "id" },
  "category", { amount: sum({ sql: "qty * price" }) });
const v = canvas(640, 460);
const bars = v.bar(revenue, {
  ...bandX("category", { paddingInner: 0.15 }).pick("x", "w"),
  ...barY("amount", { zero: true }).pick("y", "h"),   // zero anchors baseline so small bars have a box
  fill: ordinal("category"),
});
v.text(revenue, {
  ...bars.via(eq("category", "category")).pick({ x: "x", y: "y", w: "w", h: "h" }),
  ...fitText({ rotate: true }).pick("fontSize", "rotate"),
  text: field("category"), anchor: "middle", baseline: "middle", fill: "#fff",
});
return v;

Uniform typography instead of per-box sizes: fitText({ shared: true }) solves ONE font size across all labels (capped at 14, no rotation). Pick just fontSize.

const revenue = db.table("orders").rollup(
  db.table("products"), { left: "product_id", right: "id" },
  "category", { amount: sum({ sql: "qty * price" }) });
const v = canvas(640, 460);
const rects = v.rect(revenue, {
  ...treemap("amount").pick("x", "y", "w", "h"),
  fill: ordinal("category"), stroke: "#fff",
});
v.text(revenue, {
  ...rects.via(eq("category", "category")).pick({ x: "x", y: "y", w: "w", h: "h" }),
  ...fitText({ shared: true }).pick("fontSize"),
  text: field("category"), anchor: "middle", baseline: "middle", fill: "#333",
});
return v;

Multiline labels: fitText can wrap long strings into lines, which the mark draws stacked. Pick lines alongside fontSize.

const revenue = db.table("orders").rollup(
  db.table("products"), { left: "product_id", right: "id" },
  "category", { amount: sum({ sql: "qty * price" }) });
const v = canvas(640, 460);
const rects = v.rect(revenue, {
  ...treemap("amount").pick("x", "y", "w", "h"),
  fill: ordinal("category"), stroke: "#fff",
});
v.text(revenue, {
  ...rects.via(eq("category", "category")).pick({ x: "x", y: "y", w: "w", h: "h" }),
  ...fitText({ wrap: true }).pick("fontSize", "lines"),
  text: field("category"), anchor: "middle", baseline: "middle", fill: "#333",
});
return v;
