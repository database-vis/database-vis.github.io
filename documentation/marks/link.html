link

The link mark draws a stroked segment from (x1, y1) to (x2, y2). Unlike rule (which carries both endpoints on one row), a link's endpoints are almost always borrowed from other plots with via(): the position of a node in one plot becomes an endpoint here. That makes link DVL's connector primitive — the edge of a node-link diagram, the strand of a parallel-coordinates plot, the arc of an FK join — drawn from PLACED positions, never from coordinates computed in SQL.

// FK join: each link starts at a solved dot position (borrowed via a join predicate) and
// ends at its own coordinates.
const highlights = db.table("penguins").top(1, { col: "body_mass_g", desc: true }, { partition: ["species"] });
const v = canvas(640, 460);
const dots = v.dot("penguins", {
  x: linear("bill_length_mm"), y: linear("body_mass_g"), fill: ordinal("species"),
}).axis("x").axis("y");
v.link(highlights, {
  ...dots.via(eq("highlights.species", "penguins.species")).pick({ x1: "x", y1: "y" }),
  x2: linear("bill_length_mm"), y2: linear("body_mass_g"),
  stroke: "#e15759",
});
return v;

A scatter with a red link from each species' heaviest bird back to the corresponding dot.

Options

link draws drawSegment("x1", "y1", "x2", "y2"). If x2/y2 are omitted they fall back to x1/y1 (a degenerate zero-length segment). Every endpoint is affine — but the idiomatic source of an endpoint is a via().pick({...}) spread, not a literal.

Channel Type Default Meaning
x1 positional-x — start x
y1 positional-y — start y
x2 positional-x x1 end x
y2 positional-y y1 end y
stroke color — line colour (scale binding or literal)
strokeWidth pixel 1 line thickness in px
strokeDasharray text — SVG dash pattern
opacity number 1 overall opacity
strokeOpacity number 1 stroke-only opacity
title text — per-segment hover tooltip

The via() pattern

plot.via(pred, opts?) returns a borrowed reference to another plot's solved rows, joined by pred (an eq, fk, or path). .pick({ x1: "x", y1: "y" }) maps that plot's output columns (here its solved pixel x/y) onto THIS mark's channels. Spread two vias — one per endpoint — to connect a row's placement in one plot to its placement in another:

...axes[i].via(eq("id", "id")).pick({ x1: "x", y1: "y" }),
...axes[i + 1].via(eq("id", "id")).pick({ x2: "x", y2: "y" }),

pick accepts an OBJECT (rename: borrowed x → channel x1) or bare strings (same name). Pass a resolve in the via's second argument to COMPUTE an endpoint from the borrowed box (e.g. a box's midpoint) rather than take a solved column verbatim.

Examples

Parallel coordinates. One normalized dot plot per attribute at a fixed x; then a link per tuple between adjacent axes, both endpoints borrowed on the shared tuple key.

const p = db.table("penguins").withKey("id");
const attrs = ["bill_length_mm", "bill_depth_mm", "flipper_length_mm", "body_mass_g"];
const xs = [90, 300, 500, 670];
const v = canvas(740, 460);

const axes = attrs.map((a, i) => v.dot(p, {
  x: xs[i], y: linear(a, { axis: false }),
  r: 3, fill: ordinal("species"), fillOpacity: 0.85,
  title: field("id"),               // carry the key so via() can join on it
}));

for (let i = 0; i < attrs.length - 1; i++) {
  v.link(p, {
    ...axes[i].via(eq("id", "id")).pick({ x1: "x", y1: "y" }),
    ...axes[i + 1].via(eq("id", "id")).pick({ x2: "x", y2: "y" }),
    stroke: ordinal("species"), strokeWidth: 1, strokeOpacity: 0.3,
  });
}
return v;

FK join with a coloured endpoint. Same shape as the opener, but the link inherits the dot's categorical colour by re-binding the fill scale on x2.

const highlights = db.table("penguins").top(1, { col: "body_mass_g", desc: true }, { partition: ["species"] });
const v = canvas(640, 460);
const dots = v.dot("penguins", {

}).axis("x").axis("y");
v.link(highlights, {
  ...dots.via(eq("highlights.species", "penguins.species")).pick({ x1: "x", y1: "y" }),
  x2: linear("bill_length_mm"), y2: linear("body_mass_g"),
  stroke: ordinal("species"), strokeWidth: 2,
});
return v;

Node-link diagram over a force layout. Force places the member nodes; each edge borrows its two endpoints from the node plot through the declared FK, computing each node's CENTER with a resolve.

const v = canvas(760, 420);
const nodes = v.rect("members", {
  ...force("follows").pick("x", "y"),
  w: 92, h: 64, fill: "#fff", stroke: ordinal("community"), strokeWidth: 2,
});
const center = { resolve: { cx: { fn: "avg", arg: expr`f.x + 46` },
                            cy: { fn: "avg", arg: expr`f.y + 32` } } };
v.link("follows", {
  ...nodes.via(eq("follows.src", "members.id"), center).pick({ x1: "cx", y1: "cy" }),
  ...nodes.via(eq("follows.dst", "members.id"), center).pick({ x2: "cx", y2: "cy" }),
  stroke: "#b6bdc9",
});
return v;

Owner vs within partitioning. In a faceted force diagram, a top-level link with frame: opts OUT of the nest's partition (all edges, assigned to the src's panel), drawn under a nested link that keeps only within-panel edges.

const communities = db.table("members").groupby(["community"], {});
db.declareFK({ fromTable: "members", fromCols: ["community"], toTable: communities.name, toCols: ["community"] });
const v = canvas(760, 360);
const panel = v.rect(communities, { ...eqX("community").pick("x", "w"), y: 12, h: 330, fill: "#f5f6fa" });
const np = panel.nest("members", { on: eq("community", "community") }).dot({
  ...force("follows").pick("x", "y"), r: 6, fill: ordinal("community"),
});
const src = np.via(eq("follows.src", "members.id"));
v.link("follows", {                        // owner: every edge, in its src's panel
  ...src.pick({ x1: "x", y1: "y" }),
  ...np.via(eq("follows.dst", "members.id")).pick({ x2: "x", y2: "y" }),
  stroke: "#dfe3ec",
}, { frame: src });
panel.link("follows", {                    // within: σ derived from the FK catalog
  ...np.via(eq("follows.src", "members.id")).pick({ x1: "x", y1: "y" }),
  ...np.via(eq("follows.dst", "members.id")).pick({ x2: "x", y2: "y" }),
  stroke: "#b6bdc9",
});
return v;
