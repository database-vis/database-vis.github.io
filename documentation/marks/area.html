area

The area mark turns an ordered point set into a filled band. Like line/path it is an AGGREGATE: its N rows collapse to a single filled <path> per series (an array_agg over the sorted points, in SQL). The band runs along the top edge (x, y) forward, then back along the baseline (x, y2), and closes into a filled region. Points sort by the order channel, which defaults to x. Bind y2 to a column for a ribbon (a data-driven lower edge) or to a constant for a floor. A stroke (or fill) colour splits the input into one band per series.

// The top edge (x, y) forward, then the baseline (x, y2) back, closed into a filled band.
// The series is modelled inline (avg mass per species-year); y2 is a sloped baseline here.
const series = db.table("penguins").groupby(["species", "year"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
v.area(series, {
  x: linear("year"),
  y: linear("mass"),
  y2: linear("year"),   // a baseline — use a column (ribbon) or a constant (floor)
  fill: "#9ecae1",
  stroke: "#4e79a7",
}).axis("x").axis("y");
return v;

A filled band between the mass curve and the y2 baseline.

Options

area is the one aggregate that is filled: it binds the full fill paint group (both fill and stroke). It carries an extra baseline pair — x2/y2 — so the region is the area between two edges. Distinct fill/stroke values partition into series; points sort by order.

Channel Type Default Meaning
x positional-x — top-edge x position
y positional-y — top-edge y position
x2 positional-x x baseline x (usually left equal to x)
y2 positional-y — baseline y — the lower edge (column = ribbon, constant = floor)
order number x the SEQUENCE column — points sort by this before aggregating
fill color — band fill; distinct values split into one band per series
stroke color — outline colour
strokeWidth pixel 1 outline thickness (px)
strokeDasharray text — outline dash pattern
opacity number 1 whole-mark alpha
fillOpacity number 1 fill alpha
strokeOpacity number 1 outline alpha
title text — per-band <title> hover tooltip

order is an EXPRESSION channel — field("t") (aka col("t")) names a raw column; a bare string is a literal. Positionally, y2 is the baseline: leave x2 unbound and it tracks x.

Examples

Area to a constant floor — the classic filled curve down to a pixel baseline:

const yearly = db.table("penguins").groupby(["year"], { mass: avg("body_mass_g") });
const v = canvas(480, 320);
v.area(yearly, {
  x: linear("year"),
  y: linear("mass"),
  y2: 320,               // constant floor (pixel literal, bottom of the canvas)
  fill: "#cfe2f3",
  stroke: "#4e79a7",
}).axis("x").axis("y");
return v;

Ribbon — bind y2 to a column so the band has a data-driven lower edge (here min→max mass per year):

const ribbon = db.table("penguins").groupby(["year"],
  { lo: min("body_mass_g"), hi: max("body_mass_g") });
const y = linear();      // one domain for both edges (union of lo and hi)
const v = canvas(480, 320);
v.area(ribbon, {
  x: linear("year"),
  y: y("hi"),            // upper edge
  y2: y("lo"),           // lower edge — a true ribbon
  fill: "#d9e6f2", fillOpacity: 0.7, stroke: "#4e79a7",
}).axis("x").axis("y");
return v;

Multi-series bands by fill, with a legend — one band per species:

const series = db.table("penguins").groupby(["species", "year"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
const p = v.area(series, {
  x: linear("year"),
  y: linear("mass"),
  y2: 460,
  fill: ordinal("species"),
  fillOpacity: 0.5,
});
p.axis("x").axis("y").legend("fill", { placement: "right" });
return v;

Time-ordered ribbon — sequence the edges by an explicit order channel so the band follows time, not x:

const track = db.derive("track", `
  SELECT * FROM (VALUES (1,0,2,6),(2,10,1,5),(3,5,3,9))
  AS t(step, xpos, lo, hi)`);
const y = linear();
const v = canvas(320, 260);
v.area(track, {
  x: linear("xpos"), y: y("hi"), y2: y("lo"),
  order: field("step"),          // walk the edges in step order
  fill: "#9ecae1", stroke: "#4e79a7",
}).axis("x").axis("y");
return v;

Area under a line — layer area (fill) then line (crisp top edge) over shared scales:

const yearly = db.table("penguins").groupby(["year"], { mass: avg("body_mass_g") });
const x = linear(), y = linear();
const v = canvas(480, 320);
v.area(yearly, { x: x("year"), y: y("mass"), y2: 320, fill: "#e8eef5" });
v.line(yearly, { x: x("year"), y: y("mass"), stroke: "#4e79a7", strokeWidth: 2 })
 .axis("x").axis("y");
return v;
