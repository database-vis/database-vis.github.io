Marks

A mark is a channel list plus a draw function — nothing more; position intelligence belongs to scales and layouts. Each mark has its own reference page (linked below); this table is the at-a-glance index. Channels marked positional participate in the frame; pixel channels are raw pixel values; style channels pass to the pen.

Positional channels accept scales, layout outputs, literals (pixels), or via-references. The sequence marks (line/path/area/connect) also take an order channel — the column the points sort by (default x), so a trajectory whose x is not monotone still draws in sequence.

Mark Channels Notes
dot x, y, r, shape, fill… circles; r in pixels (default 3)
rect x, y, x2, y2, w, h, fill, stroke… container-eligible (can host nested plots)
bar same as rect container-eligible; pairs with bandX/barY layouts
cell x, y, fill… grid cell; container-eligible
line x, y, order, stroke… aggregate: N rows → one path per series (order sets sequence, default x)
path x, y, order, stroke… the aggregate primitive — an ordered point set → one open polyline (line = path); bind order for tracks/trajectories
area x, y, x2, y2, order, fill… aggregate: band between two y's, ordered
connect x, y, order, stroke… N points → N−1 segments between successive points (a LEAD window per series, ordered by order); the pair sibling of line
rule x, y, x2, y2, stroke… a segment (reference lines, underlines)
link x1, y1, x2, y2, stroke… a connector; endpoints usually via-references
bezier x1, y1, x2, y2, curvature, cx1…cy2, stroke… a curved connector (cubic Bézier); link's channels + a bend
arrow x1, y1, x2, y2, headType, headSize, stroke… directed link
vector x, y, length, angle, stroke… length px, angle radians (0 = +x)
tick x, y, len, orient, stroke… short axis-style tick
text x, y, dx, dy, text, anchor, baseline, fontSize, rotate, fill, opacity text: takes an expression — a bare string is a literal (use col("name") for column values)
image x, y, w, h, src, opacity <image href> box

Each per-mark page follows the shared shape: model → minimal example → options/channels → many worked examples → see also.

Adding your own mark

A mark is a descriptor — a channel list plus a draw — built with defineMark. The renderer never changes; adding a mark is purely local.

import { defineMark } from "$lib/dvl/ir/mark";
import { num, styleOf } from "$lib/dvl/ir/pen";

const wedge = defineMark({
  type: "wedge",
  channels: { x: "positional-x", y: "positional-y", r: "pixel", angle: "number", fill: "color", stroke: "color" },
  pixelAxes: { r: "xy" },   // r scales on both axes
  draw: (row, pen) => pen.path(wedgePath(num(row, "x"), num(row, "y"), num(row, "r"), num(row, "angle")), styleOf(row)),
});
  • channels maps each channel name to a kind — positional-x/positional-y (join the frame), pixel (raw pixels), and color/shape/text/number (style). The kind type-checks what a binding may be: a scale, a layout output, a literal, or a via-reference.
  • draw(row, pen) is the only place geometry is emitted. The row is fully resolved (pixels, colours), and the Pen (circle/rect/line/path/text/…) is substrate-agnostic — the same draw feeds the SVG renderer and the bounding-box measurer.
  • Opt into behaviours with cardinality: "aggregate" (N rows → one path, like line) or containerEligible: true (can host nested plots, like rect).

Register the descriptor, then bind it — the built-in v.dot/v.rect/… are sugar over the generic v.mark:

registries.marks.register(wedge.type, wedge);
v.mark("wedge", "penguins", { x: linear("bill_length_mm"), y: linear("body_mass_g"), r: 6, angle: field("year") });
