bezier

The bezier mark is link's curved sibling: one cubic Bézier per row from (x1, y1) to (x2, y2). It carries link's exact endpoint contract — endpoints are almost always borrowed from other plots with via() — and adds the curve's shape on top: a curvature bend, or explicit control points. Reach for it where straight edges overlap or double back — node-link diagrams with reciprocal edges, arc diagrams, flow maps.

const v = canvas(640, 460);
const np = v.dot("members", { ...force("follows").pick("x", "y"), r: 6, fill: ordinal("community") });
v.bezier("follows", {
  ...np.via(eq("follows.src", "members.id")).pick({ x1: "x", y1: "y" }),
  ...np.via(eq("follows.dst", "members.id")).pick({ x2: "x", y2: "y" }),
  curvature: 0.3,
  stroke: "#b6bdc9",
});
return v;

A force-placed node-link diagram whose edges bow 0.3 of their length to the side — reciprocal edges (a→b and b→a) bow to opposite sides automatically, because curvature is signed by the segment's direction.

Options

bezier draws M x1,y1 C cx1,cy1 cx2,cy2 x2,y2 — stroke-only (fill defaults to none, like line/path). If x2/y2 are omitted the chord is zero-length and nothing is drawn. The two control points come from ONE of:

  • curvature (the default path): a signed perpendicular bow as a fraction of the chord length. The controls sit at ⅓ and ⅔ along the chord, offset curvature × chord to the normal side; the curve's peak deviation is ≈ ¾ of that. 0 is a straight segment; negative bends the other way; the sign flips with the segment direction, so reciprocal edges separate.
  • cx1/cy1/cx2/cy2: explicit control points, ordinary positional channels — scale bindings, layout outputs, or via() borrows, placed by the frame affine like any endpoint. When all four are bound they win over curvature (edge bundling, flow maps).
Channel Type Default Meaning
x1 positional-x — start x
y1 positional-y — start y
x2 positional-x x1 end x (missing ⇒ degenerate, not drawn)
y2 positional-y y1 end y
curvature number 0.25 signed bow, fraction of chord length
cx1, cy1 positional derived first control point (with cx2/cy2, overrides curvature)
cx2, cy2 positional derived second control point
stroke color — curve colour (scale binding or literal)
strokeWidth pixel 1 curve thickness in px
strokeDasharray text — SVG dash pattern
opacity number 1 overall opacity
strokeOpacity number 1 stroke-only opacity
title text — per-curve hover tooltip

Examples

Arc diagram. Nodes on a line; every edge is a semicircle-ish arc above it. One fixed y, so each chord is horizontal and curvature lifts every arc the same way.

const v = canvas(740, 300);
const np = v.dot("members", { x: point("id"), y: px(240), r: 4, fill: ordinal("community") });
v.bezier("follows", {
  ...np.via(eq("follows.src", "members.id")).pick({ x1: "x", y1: "y" }),
  ...np.via(eq("follows.dst", "members.id")).pick({ x2: "x", y2: "y" }),
  curvature: 0.5,
  stroke: ordinal("src"), strokeOpacity: 0.5,   // color each arc by its source node (a column of `follows`)
});
return v;

Boundary attachment + reciprocal separation. bezier composes with the route() layout exactly like link: route owns the two endpoint borrows, clips each to its node's boundary (a dot's circle, a rect's edge), and hands the clipped x1,y1,x2,y2 to the curve — which then bows from the boundary points. Reciprocal edges (a→b and b→a) still separate automatically, because signed curvature flips with the reversed chord direction. Options ride route's second argument: pad stands the endpoints a few px further off the boundary, shape/shape1/shape2 override the metric (default = the referent mark's declared boundary — ellipse for dots, box for rects).

const v = canvas(640, 460);
const np = v.dot("members", { ...force("follows").pick("x", "y"), r: 6, fill: "#4e79a7" });
v.bezier("follows", {
  ...route({ src: np.via(eq("follows.src", "members.id")),
             dst: np.via(eq("follows.dst", "members.id")) }, { pad: 2 }).pick("x1", "y1", "x2", "y2"),
  stroke: "#999", strokeWidth: 1.5,
});
return v;

Explicit control points — a hand-shaped flow. Bind the controls to data (or literals): here every curve leaves its source horizontally and enters its target horizontally, Sankey-style. The aligned column x's are pixel literals; the varying y's ride one linear scale per side (reused on the control point so it shares the endpoint's height — a flat departure/arrival).

const flows = db.derive("flows", `
  WITH agg AS (SELECT island, species, count(*) AS n FROM penguins GROUP BY 1, 2)
  SELECT island, species, n,
         dense_rank() OVER (ORDER BY island)  AS row_from,
         dense_rank() OVER (ORDER BY species) AS row_to
  FROM agg`);
const v = canvas(640, 460);
const yf = linear("row_from"), yt = linear("row_to");
v.bezier(flows, {
  x1: px(100), y1: yf,  x2: px(540), y2: yt,          // aligned columns: x is a pixel literal
  cx1: px(320), cy1: yf, cx2: px(320), cy2: yt,        // controls at mid-x, each level with its endpoint
  stroke: ordinal("island"), strokeWidth: px("n"), strokeOpacity: 0.6,  // ribbon per island, width = count
});
return v;

Data-driven bend. curvature is an ordinary number channel — bind a column to bend heavier flows harder.

// synthesize a per-edge weight (follows carries only src/dst) → a signed bend in [-0.5, 0.5]
const edges = db.derive("wedges", `SELECT *, ((src * 7 + dst) % 11) / 10.0 - 0.5 AS weight FROM follows`);
const v = canvas(640, 460);
const np = v.dot("members", { ...force("follows").pick("x", "y"), r: 6, fill: "#4e79a7" });
v.bezier(edges, {
  ...np.via(eq("wedges.src", "members.id")).pick({ x1: "x", y1: "y" }),
  ...np.via(eq("wedges.dst", "members.id")).pick({ x2: "x", y2: "y" }),
  curvature: field("weight"),
  stroke: "#999",
});
return v;

See also

  • link — the straight sibling; same endpoint channels and via() idiom. Both marks get boundary attachment through route(): it clips each endpoint to the node's edge, then link draws the segment and bezier bows the curve from those clipped points.
  • arrow — a directed straight connector with a head.
  • connect — successive points within a group, via a LEAD window.
  • force — the node placement curved edges usually connect.
