The frame

Where pixels come from: operators emit normalized [0, 1] coordinates; a constraint solver arranges boxes (canvas, panels, axis gutters, margins); the affine maps normalized → pixel per plot and axis. Axes are ordinary plots over a scale's tick relation — placement, merging of duplicate sibling axes, tick-count-from-extent, and label measurement all flow through the same dataflow as your marks. Covers .axis(), axis options, margins/insets, and pixel-passthrough channels for precomputed geometry.

Normalized to pixel

No scale or layout ever computes a pixel. Scales apply a per-row normalization and producer layouts emit normalized [0, 1] positions; the frame does the rest:

  1. the solver arranges the boxes — the canvas, panels, axis gutters, and margins — as constraints, so a gutter's width comes from its measured labels, not a guess;
  2. the affine maps each plot's (and each axis's) normalized coordinates into its solved box — the unique normalized → pixel step;
  3. measurement reads placed geometry back as size demands (label extents, content boxes), which re-enter the solver until the arrangement settles.

Pixel channels skip this pipeline: a positional literal is already a pixel, so precomputed geometry passes straight through to the placed box.

Axes are ordinary plots

An axis is not special-cased rendering: it is a plot over the scale's tick relation, flowing through the same dataflow as your marks. That one design carries the rest — placement is a solved box like any panel, duplicate sibling axes merge, the tick count derives from the solved extent, and label measurement feeds the gutter width back into the solver.

// axes are ordinary plots over the scale's tick relation: pin a tick count on x, place
// the y axis on the RIGHT — the gutters are solved from measured labels, not guessed.
const v = canvas(640, 320);
v.dot("penguins", { x: linear("flipper_length_mm"), y: linear("body_mass_g"), fill: "#4e79a7" })
 .axis("x", { ticks: { count: 4 } })
 .axis("y", { placement: "right" });
return v;

Axis options

Axis options — pass them to the .axis(dim, opts) directive or scale-level as linear("c", { axis: {…} }) (the directive wins on conflict; axis: false suppresses the axis):

Option Type Default Meaning
title boolean | string | object derived axis title — DEFAULT-ON: a data axis titles itself with the scale's column (or the authored expression); false or "" suppresses; a string sets the text; an object styles the text mark (text, fontSize 11, fill/color #333, opacity, dx/dy nudge, rotate, anchor, baseline). A synthetic axis with no column derives nothing and stays title-less
ticks number | { count, algorithm } auto tick count, or a count + a registered tick algorithm
tickLength number 5 tick-mark length; label nudges and the gutter pad derive from it
fontSize number 10 tick-label font size
placement "top"|"right"|"bottom"|"left" auto pin the axis to a side (the .axis() directive; scale-level uses side)
place "inside" | "outside" role rank order among co-located bands on the same side
gap number 0 px before the band along the stack normal
pad number | per-side 0 inner padding between the band edges and its ticks/labels 
// axis TITLES default ON — each data axis names its column. A string overrides the derived
// text; `title: false` (or "") turns one off.
const v = canvas(640, 340);
v.dot("penguins", { x: linear("flipper_length_mm", { axis: { title: false } }), y: linear("body_mass_g"), fill: "#4e79a7" })
 .axis("y", { title: "body mass (g)" });
return v;
