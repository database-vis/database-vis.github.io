Plots and marks

A plot is one mark type over one source relation; a canvas hosts plots and solves their shared frame. Marks are deliberately simple — a channel list and a draw function — so all intelligence about position lives in scales and layouts. Covers: canvas, the mark methods, layering order, containers (rect/bar/cell can host nested plots), and params.

// three layers over one canvas share a frame: the cloud, a per-species highlight
// (modelled inline on the db branch), and labels riding the highlight's scales.
const heaviest = db.table("penguins").top(1, { col: "body_mass_g", desc: true }, { partition: ["species"] });
const v = canvas(640, 360);
const x = linear("bill_length_mm"), y = linear("body_mass_g");
v.dot("penguins", { x, y, fill: "#cfd3dc" });
v.dot(heaviest, { x, y, r: 5, fill: "#e15759" });
v.text(heaviest, { x, y, text: field("species"), dx: 8, anchor: "start", fontSize: 10, fill: "#333" });
return v;

The canvas

canvas(width, height) creates the root container. It owns the pixel box every plot shares: plots emit normalized positions, and the canvas's frame solves the arrangement and maps them to pixels (see The frame). All plots added to one canvas participate in the same constraint solve, so their axes, gutters, and panels are laid out together. A DVL program ends with return v; — the canvas is the value that renders.

The mark methods

Each mark is a method on the canvas (or on a container plot): v.<mark>(source, channels). The source is a table name or a derived relation; the channels object binds each of the mark's declared channels to a value — a scale, a layout output via .pick(), a literal, an expression, or a cross-mark reference (see Channels).

The built-in marks — each has its own reference page under marks/:

  • Point-like: dot (circles, r in pixels), cell (grid cell), image, text.
  • Box-like: rect, bar — the container-eligible family together with cell.
  • Sequence marks (N rows → paths/segments per series): line, path, area, connect — these also take an order channel (default x) that sets point sequence.
  • Segments and connectors: rule, link, bezier, arrow, vector, tick.

A mark method returns a plot handle. The handle carries the per-plot directives seen throughout these pages: .axis(dim, opts) and .legend(channel) attach guides, .via(path) lets other plots borrow this plot's solved positions, and container plots add nested layers with further mark calls (below).

Layering order

Layers draw in call order: the first plot added to a canvas renders first (bottom), later plots render on top. In the example above the gray cloud is called first, the red highlights second, and the labels last — so the labels sit above both.

Containers

rect, bar, and cell can host nested plots: a mark call chained on a container's handle draws inside each container instance rather than on the canvas. With a nest key this is how small multiples work — see Facets and nesting and Subplots.
