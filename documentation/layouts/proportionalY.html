proportionalY

proportionalY sizes a stack of segments vertically in proportion to a value column: each row's height is its share of the total, and each row's y is the running sum of the heights above it. It emits y (segment top) and h (segment height), both normalized to [0, 1] as cumulative shares of the sum (y = cumsum(preceding) / total, h = value / total). It is the vertical twin of proportionalX — a single-column percent stack.

const revenue = db.table('orders').rollup(
  db.table('products'), { left: 'product_id', right: 'id' },
  'category', { amount: sum({ sql: 'qty * price' }) });
const v = canvas(200, 460);
v.rect(revenue, {
  ...proportionalY('amount').pick('y', 'h'),
  x: px(40), w: 120,
  fill: ordinal('category'),
  stroke: '#fff',
});
return v;

A single vertical bar split into category segments whose heights are proportional to revenue — a 100%-stacked column. The measure is modelled inline with rollup.

Options

proportionalY(value) takes a single numeric input. There is no trained domain — outputs are cumulative shares in [0, 1] (the same rationale as stackY), so nothing is tickable. Negative values clamp to 0; a zero total falls back to equal shares (i / n, 1 / n).

Input Type Meaning
value double the measure whose magnitude sets each segment's height

.pick() outputs:

Output Space Meaning
y relative coord y segment top = cumsum(preceding value) / total
h relative len y segment height = value / total

Bind by spreading: ...proportionalY('amount').pick('y', 'h'). Segment order down the column follows the input relation's order. No options bag.

Examples

Categories by revenue, a 100%-stacked column (as above):

const revenue = db.table('orders').rollup(
  db.table('products'), { left: 'product_id', right: 'id' },
  'category', { amount: sum({ sql: 'qty * price' }) });
const v = canvas(200, 460);
v.rect(revenue, { ...proportionalY('amount').pick('y', 'h'), x: px(40), w: 120,
                  fill: ordinal('category'), stroke: '#fff' });
return v;

Penguin counts per species as a proportional column — group, then size by count:

const bySpecies = db.table('penguins').groupby(['species'], { n: count() });
const v = canvas(160, 400);
v.rect(bySpecies, { ...proportionalY('n').pick('y', 'h'), x: px(40), w: 80,
                    fill: ordinal('species'), stroke: '#fff' });
return v;

Label each segment at its midpoint via a computed-center via, fitted with fitText:

const revenue = db.table('orders').rollup(
  db.table('products'), { left: 'product_id', right: 'id' },
  'category', { amount: sum({ sql: 'qty * price' }) });
const v = canvas(260, 460);
const bars = v.rect(revenue, { ...proportionalY('amount').pick('y', 'h'), x: px(40), w: 140,
                              fill: ordinal('category'), stroke: '#fff' });
v.text(revenue, {
  ...bars.via('category').pick({ x: 'x', y: 'y', w: 'w', h: 'h' }),
  ...fitText(),
  text: field('category'), anchor: 'middle', baseline: 'middle', fill: '#fff',
});
return v;

Small multiples of proportional columns — one column per species panel, split by eqX, each column filled by its own proportionalY:

const revenue = db.table('orders').rollup(
  db.table('products'), { left: 'product_id', right: 'id' },
  'category', { amount: sum({ sql: 'qty * price' }) });
const panels = db.table('penguins').groupby(['species'], {});
const v = canvas(600, 400);
v.rect(panels, { ...eqX('species').pick('x', 'w'), y: px(10), h: 380, fill: '#f5f6fa' })
 .rect(revenue,
  { ...proportionalY('amount').pick('y', 'h'), x: px(8), w: 60, fill: ordinal('category'), stroke: '#fff' });
return v;
