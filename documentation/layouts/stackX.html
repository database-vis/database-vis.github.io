stackX

stackX is a layout operator — the x-mirror of stackY. It cumulates weighted segments within a group and normalizes them to fill an extent horizontally: each mark occupies [cumsum − weight, cumsum] of its group's total, so x (segment LEFT) = cumsum/total and w (segment width) = weight/total. Bind both with .pick("x","w"). The sums are WINDOWED (so it lowers to SQL); the relative [0,1] shares let the frame STRETCH each group to fill its box — a percent-stack per group. Pair it with bandY to get horizontal stacked bars.

const stacks = db.table("penguins").groupby(["species", "island"], { n: count() });
const v = canvas(640, 460);
v.rect(stacks, {
  ...bandY("species", { paddingInner: 0.2 }).pick("y", "h"),
  ...stackX({ weight: "n", group: "species", order: "island" }).pick("x", "w"),
  fill: ordinal("island"),
}).axis("y");
return v;

Horizontal stacked bars: bandY places one bar per species; stackX partitions each bar's total by island and cumulates the segments to fill the width.

Options & Outputs

Apply-only (no trained fit) — outputs are within-group relative shares in [0,1]. Constructor takes one bag of input columns: weight (required — segment magnitude), group (the partition), order (the stacking sequence; default = weight).

output space meaning
x relative coord (x) segment LEFT (cumsum / total)
w relative len (x) segment width (weight / total)

See also

  • stackY — the y-mirror; grouped stacked segments → {y,h} (vertical stacked bars).
  • bandY — the categorical placement each horizontal stacked bar sits in.
