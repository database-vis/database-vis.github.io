grid-stack

grid-stack is a refine/placement layout: it packs already-materialized items into absolute pixel boxes by stacking them row- or column-major in a gutter (or as an inside overlay). It reads the content box's four extent cells at execution and emits x, y, w, h per item. Its distinguishing feature is that the item pitch is MEASURED from the real text extents of the labels — the widest label sets the column width — so items never overlap regardless of label length.

This is the layout behind the auto-legend. You almost never author it directly: .legend(channel) lowers to a vtable + apply-only scale + a grid-stack (item boxes) + swatch/text plots + a gutter reservation. It is documented here because it is the engine that sizes and arranges legend items.

const v = canvas(640, 480);
v.dot("penguins", {
  x: linear("bill_length_mm"), y: linear("body_mass_g"),
  fill: ordinal("species"),
}).axis("x").axis("y").legend("fill", { placement: "right" });
return v;

The .legend("fill", …) produces a right-gutter legend: one swatch + text row per species, each row sized to the measured label width (Adelie, Chinstrap, Gentoo), stacked at that measured pitch. The grid-stack is what lays those rows out.

The measured-pitch model — vs pitchY

There are two item-stack layouts, and they differ in one axis:

  • grid-stack packs in 2-D, reads the content box, and measures the item width from the actual (optionally truncated) labels at execution. It is JS-only because it reads extent cells and calls the measurement provider. Use it when items have variable-width text and must not overlap.
  • pitchY is a 1-D fixed-pitch stack: y = pad + rank·(pitch+gap), height = pitch, independent of the content box. It needs no box and lowers to pure SQL (a windowed rank). Use it when the pitch is known up front.

The legend uses grid-stack precisely because label widths aren't known until measured; itemW is only a seed/fallback used when no measurement provider is available.

Options

grid-stack is a plan-only operator — it has no authoring factory. Its options are populated by the legend-lowering pass (lower-legends.ts) from .legend(channel, opts). The outputs and the GridStackOpts shape:

Outputs (x, y, w, h)

output space meaning
x pixel(coord x) box origin x (+ inset.x)
y pixel(coord y) box origin y (+ inset.y)
w pixel(len x) item width — the measured pitch (uniform across items)
h pixel(len y) item height (itemH)

GridStackOpts

option type default meaning
contentEdges {left,right,top,bottom} cells — the content box read at execution to size/anchor the grid
itemW number — seed item width; the fallback pitch when no measure provider is present
itemH number — item height (fixed)
pad number 4 inset from the anchor edge
major "column" | "row" side-derived wrap axis. left/right ⇒ column (stack down); top/bottom ⇒ row (stack across)
side "left"|"right"|"top"|"bottom" "right" which gutter the grid occupies (determines the anchor)
inside "top-left"|"top-right"|"bottom-left"|"bottom-right" — an overlay in a content-box corner; consumes no layout space; side is ignored
label {column, fontSize, family?, base, k?} — measure item width from REAL text extents: w = base + max measured label width. k truncates (LEFT(label,k)) before measuring
inset {x, y} {0,0} glyph anchor inside each box; the emitted x/y shift by this, w/h stay box-sized

.legend(channel, opts) — the authoring surface

option maps to
placement: left|right|top|bottom|inside (alias position) side / inside (which gutter or corner)
swatchSize swatch diameter
fontSize label font size (also feeds the measure)
pad, itemGap item padding / gap
cols, major wrap capacity / axis
maxLabelChars truncation k for the measure

An unknown placement value throws (silent typos cost a debugging session).

Examples

A right-gutter categorical legend

The default placement. One row per category, packed column-major (stacked down), pitch measured from the species names:

const v = canvas(640, 480);
v.dot("penguins", {
  x: linear("bill_length_mm"), y: linear("body_mass_g"), fill: ordinal("species"),
}).axis("x").axis("y").legend("fill", { placement: "right" });
return v;

Top placement — items stack across (row-major)

A top/bottom gutter wraps row-major, and the whole grid grows AWAY from the plot (a second row climbs up, never onto the content):

v.dot("penguins", { x: linear("bill_length_mm"), y: linear("body_mass_g"), fill: ordinal("species") })
 .legend("fill", { placement: "top" });

Inside overlay

placement: "inside" anchors the grid in a content-box corner, consuming no layout space:

v.dot("penguins", { x: linear("bill_length_mm"), y: linear("body_mass_g"), fill: ordinal("species") })
 .legend("fill", { placement: "inside" });

A size legend

Non-fill channels work too; size/opacity map to the swatch's r/fillOpacity, with a neutral swatch fill:

v.dot("penguins", { x: linear("bill_length_mm"), y: linear("body_mass_g"), size: linear("body_mass_g") })
 .legend("size", { placement: "right" });

Bound long labels with maxLabelChars

Truncate labels before measuring so a very long category name doesn't blow out the gutter width — the pitch is base + max(LEFT(label, k)):

v.dot("orders", { x: linear("qty"), y: linear("year"), fill: ordinal("cust_id") })
 .legend("fill", { placement: "right", maxLabelChars: 12 });
