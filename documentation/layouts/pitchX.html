pitchX

pitchX is a layout operator — the x-mirror of pitchY. It lays one fixed-width band per row across a row, ranked by an order column — the SQL-pushable horizontal legend/list stack. It emits x (the band LEFT) and w (the band width), bound with .pick("x","w"). Unlike stackX, whose relative shares the frame STRETCHES to fill an extent, pitchX's outputs are PIXEL space and EXTENT-INDEPENDENT: the row's total width is intrinsic (N·pitch), not the box's. Being a pure windowed derive (rank), it lowers to both SQL and JS.

const bySpecies = db.table("penguins").groupby(["species"], { n: count() });
const v = canvas(380, 120);
const bands = pitchX("species", { pitch: 96, gap: 12, pad: 16 });
v.rect(bySpecies, { ...bands.pick("x", "w"), y: px(16), h: 22, fill: ordinal("species") });
v.text(bySpecies, { ...bands.pick("x", "w"), y: px(58), text: field("species"),
                    fill: "#c5cbe0", anchor: "start" });
return v;

A horizontal swatch legend: one swatch rect + one label per species, ranked by name at a constant 96px pitch. The SAME pitchX instance is .pick-ed twice, so both marks share one ranking.

Options & Outputs

Pixel offsets measured ACROSS from the row's origin: x = pad + rank·(pitch + gap), w = pitch. rank is the 0-based dense rank of the order column. Apply-only (no trained fit).

output space meaning
x pixel coord (x) band LEFT (pad + rank·step)
w pixel len (x) band width (pitch)

Options bag (2nd arg): pitch (default 16), gap (default 0), pad (default 0).

See also

  • pitchY — the y-mirror; fixed-pitch bands → {y,h} (vertical legend/list column).
  • stackX — the extent-FILLING horizontal sibling (percent stack).
