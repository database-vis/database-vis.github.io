route

route is a refine layout for edge marks: it takes the two endpoint via() handles of a link, arrow, or bezier, and rewrites each endpoint from its node's center onto the node's boundary nearest the far end. A dot referent clips on its circle (exactly r from center); a rect/bar/image referent clips on its rectangle edge. Because centers are derived from the node's box edges, the anchor convention (a centered dot, a top-left rect) never skews the clip — the edge always meets the visible shape, never dives under it.

const v = canvas(640, 460);
const nodes = v.dot("members", { ...force("follows").pick("x", "y"), r: 8, fill: ordinal("community") });
v.link("follows", {
  ...route({ src: nodes.via(eq("follows.src", "members.id")),
             dst: nodes.via(eq("follows.dst", "members.id")) }).pick("x1", "y1", "x2", "y2"),
  stroke: "#b6bdc9",
});
return v;

A force node-link diagram whose edges stop exactly on each dot's circle instead of piercing to the center. Without route, the same two via().pick() borrows land the endpoints on the centers (the raw solved positions) — which is what you want when the nodes are point-sized, and what route degrades to when a referent has no box.

The refine model — how route gets the box

Like fitText, route runs after placement: it reads pixel positions, not data. But instead of picking columns off ONE mark, it owns the two endpoint borrows itself — you pass it the src and dst via handles, not .pick()ed channels:

route({ src: nodes.via(eq("follows.src", "members.id")),
        dst: nodes.via(eq("follows.dst", "members.id")) }).pick("x1", "y1", "x2", "y2")

Each handle expands into the referent's placed anchor plus its box edges, riding the same via join — so the box travels with the borrow, no separate query. route clips the ray from each box center toward the far endpoint at the point it exits the boundary, and emits the four endpoint channels. You MUST .pick("x1", "y1", "x2", "y2") — all four — and spread them onto the edge mark.

A referent with no full box (a dot with no r is still boxed by its default radius, but a mark with no size channels, or a raw-column endpoint) degrades that endpoint to its center — mixed edges clip only the boxed end, never producing NaN or an inverted segment.

Options

route(bag, options?). The bag carries src/dst (the via handles); options tunes the clip.

.pick(...) outputs

output space meaning
x1, y1 pixel(x|y) source endpoint, clipped to the src node's boundary
x2, y2 pixel(x|y) target endpoint, clipped to the dst node's boundary

Options bag

option type default meaning
src via handle — the source node borrow (nodes.via(eq(...))) — required
dst via handle — the target node borrow — required
pad number 0 stand each clipped endpoint off the boundary by this many px along the segment (arrowhead breathing room, edge–node gap)
shape "box" | "ellipse" the referent's override the boundary metric for BOTH ends
shape1 / shape2 "box" | "ellipse" the referent's override just the src / dst end

The metric defaults to the referent mark's declared boundary: ellipse (L2) for dot, box (L∞) for rect/bar/image. Override only when the shape lies (a square dot, a circular avatar in a rect).

Examples

Rectangular nodes

Nodes with real width and height: route clips each edge to the rectangle edge nearest the far node. anchor: "center" centers each rect on its solved force position.

const v = canvas(680, 460);
const nodes = v.rect("members", {
  ...force("follows").pick("x", "y"),
  w: 70, h: 34, anchor: "center", fill: "#eef2fb", stroke: ordinal("community"), strokeWidth: 2,
});
v.link("follows", {
  ...route({ src: nodes.via(eq("follows.src", "members.id")),
             dst: nodes.via(eq("follows.dst", "members.id")) }).pick("x1", "y1", "x2", "y2"),
  stroke: "#b6bdc9",
});
return v;

Every connector terminates on a box edge — no line vanishes under a node. Add text labels borrowing the same rects to finish a boxed node-link diagram.

Curved edges over rect nodes (bezier + route)

route works with bezier exactly as with link: it clips the endpoints to the boundary, and the curve bows from those clipped points. Reciprocal edges separate because signed curvature flips with the chord direction. pad: 3 opens a small gap between each curve and the box.

const v = canvas(680, 460);
const nodes = v.rect("members", {
  ...force("follows").pick("x", "y"),
  w: 70, h: 34, anchor: "center", fill: "#eef2fb", stroke: ordinal("community"), strokeWidth: 2,
});
v.bezier("follows", {
  ...route({ src: nodes.via(eq("follows.src", "members.id")),
             dst: nodes.via(eq("follows.dst", "members.id")) }, { pad: 3 }).pick("x1", "y1", "x2", "y2"),
  curvature: 0.25, stroke: "#8894ab", strokeWidth: 1.5,
});
return v;

Arrowheads that land on the boundary

An arrow head is only useful when it stops at the node it points to. route puts the tip on the boundary; pad lifts it a couple px clear of the edge.

const v = canvas(640, 460);
const nodes = v.dot("members", { ...force("follows").pick("x", "y"), r: 9, fill: ordinal("community") });
v.arrow("follows", {
  ...route({ src: nodes.via(eq("follows.src", "members.id")),
             dst: nodes.via(eq("follows.dst", "members.id")) }, { pad: 2 }).pick("x1", "y1", "x2", "y2"),
  headType: "triangle", headSize: 8, stroke: "#8894ab",
});
return v;

Padding: an edge–node gap

pad stands both clipped ends off the boundary along the segment — a uniform gap between edges and nodes that reads as breathing room in dense graphs.

const v = canvas(640, 460);
const nodes = v.dot("members", { ...force("follows").pick("x", "y"), r: 10, fill: ordinal("community") });
v.link("follows", {
  ...route({ src: nodes.via(eq("follows.src", "members.id")),
             dst: nodes.via(eq("follows.dst", "members.id")) }, { pad: 6 }).pick("x1", "y1", "x2", "y2"),
  stroke: "#b6bdc9", strokeWidth: 1.5,
});
return v;

Override the boundary metric

A dot clips as an ellipse by default. If you drew square markers with a dot's r, tell route to exit the L∞ box instead:

...route({ src, dst }, { shape: "box" }).pick("x1", "y1", "x2", "y2"),

Use shape1 / shape2 to override just one end (a boxed source into a circular target).
