hull

hull draws a polygon around a set of points — a convex hull or a bounding box. It is not a .pick() layout like bandX/force; it is a mark fed by a geometry via resolver. The pattern is pure DVL: via() joins each group row to its member points and borrows their solved (pixel) positions, and the resolver reduces that match set to one polygon the hull mark draws. It is the resolver class of layout (see the layouts overview) — a reduction of a matched set to one shape per row, not a .pick() placement operator.

// Convex hull per species, over a bill-length × body-mass scatter.
const species = db.table('penguins').groupby(['species'], {});   // one row per species = the hull anchors
const v = canvas(640, 460);
const dots = v.dot('penguins', {
  x: linear('bill_length_mm'), y: linear('body_mass_g'), fill: ordinal('species'),
});
v.hull(species, {
  ...dots.via(on('species'), { resolve: 'convex' }).pick('path'),
  stroke: ordinal('species'), fill: ordinal('species'), fillOpacity: 0.12,
}).axis('x').axis('y');
return v;

The dot plot places the points; species is one row per group (the hull anchors). Each hull borrows its species' dots through via(on('species')) and the convex resolver reduces them to a polygon. Because the borrowed positions are already placed pixels, the hull is drawn as-is — no paired scale, no re-placement.

The resolver

A geometry resolver reduces the matched point set to a polygon — an array of [x, y] vertices in drawing order, closed by the mark. Name it as the bare resolve — { resolve: 'convex' }. The resolver declares its default output (path) and default inputs (the placed x/y channels), so neither the { path: … } wrapper nor cols is needed in the common case:

Resolver What it computes Output Substrate
convex the convex hull (Andrew's monotone chain) hull vertices; ≤2 unique points draw as-is (no degenerate triangle) SQL (spatial) or JS
bbox the axis-aligned bounding box the 4 corners enclosing every point JS

Override the inputs only when the point columns are not x/y: { resolve: { hull: { js: 'convex', cols: ['px', 'py'] } } } (the map form also renames the output). An empty match set yields no polygon.

The mark

hull is a per-row mark: one polygon per source row. Its path channel carries the polygon (bound from the via .pick('path')); it strokes and fills like a closed shape, so it takes the full paint set (stroke, fill, fillOpacity, strokeWidth, …). Position lives in the vertices, so there are no x/y channels.

Channel Kind Meaning
path geometry the polygon vertices (a via geometry resolution), drawn closed
stroke, fill, fillOpacity, strokeWidth, … paint the usual paint channels

Examples

A bounding box per species instead of a hull — same call, bbox for convex:

const species = db.table('penguins').groupby(['species'], {});
const v = canvas(640, 460);
const dots = v.dot('penguins', { x: linear('bill_length_mm'), y: linear('body_mass_g'), fill: ordinal('species') });
v.hull(species, {
  ...dots.via(on('species'), { resolve: 'bbox' }).pick('path'),
  stroke: ordinal('species'), fill: 'none', strokeWidth: 1.5,
}).axis('x').axis('y');
return v;

One hull over all points — anchor on UNIT (the built-in single-row table) and match every point with the on('*') CROSS product; no grouping or join key at all. A hull with an unfilled interior reads as an outline:

const v = canvas(640, 460);
const dots = v.dot('penguins', { x: linear('bill_length_mm'), y: linear('body_mass_g'), fill: '#4e79a7', fillOpacity: 0.6 });
v.hull(UNIT, {
  ...dots.via(on('*'), { resolve: 'convex' }).pick('path'),
  stroke: '#e15759', fill: 'none', strokeWidth: 2,
}).axis('x').axis('y');
return v;

UNIT is a one-row, one-column (value: 1) table — the anchor for a mark with no table of its own; UNIT(v) carries a different scalar.

Substrate

convex is SQL-pushed: the bare resolve: 'convex' runs ST_ConvexHull in DuckDB (the spatial extension, loaded on demand — an unavailable extension throws, it never silently falls back), returning the hull as WKT the mark parses. To force the client-side reduce instead (the JS escape hatch — e.g. no extension, offline), pass the reducer inline: { resolve: { path: { js: (matches, mark) => … } } }. bbox is JS (min/max needs no extension). Concave/α-shape hulls (ST_ConcaveHull, multi-ring paths) are deferred — see design/2026-07-20-convexhull-resolver.html.
