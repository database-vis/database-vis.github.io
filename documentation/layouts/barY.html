barY

barY is a layout operator that turns a numeric value into a vertical bar rising from the baseline. It emits TWO outputs — y (the bar top) and h (its height down to the baseline) — so you bind both with .pick("y","h"). It trains a min/max fit like a linear scale and carries nice-linear ticks, so .axis("y") is labelled. barY is the y-axis partner of bandX: bandX("cat").pick("x","w") gives {x,w}, barY("value").pick("y","h") gives {y,h}, and one rect/bar mark draws the bar.

const bySpecies = db.table("penguins").groupby(["species"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
v.bar(bySpecies, {
  ...bandX("species").pick("x", "w"),
  ...barY("mass").pick("y", "h"),   // bar top + height to baseline
  fill: ordinal("species"),
}).axis("x").axis("y");
return v;

A column chart of average body mass per species.

Options

barY emits normalized relative positions in [0,1]: y is the value's position between the domain's lo and hi, and h is the SAME ratio treated as a length (the baseline is t=0). The frame affine maps y as a coord and h as a len into the y box.

Outputs (bind with .pick(...)):

output space meaning
y relative coord (y) the bar TOP (value's position in the domain)
h relative len (y) the bar HEIGHT down to the baseline

Options bag (2nd arg):

option type default meaning
zero boolean false anchor the domain at the baseline: lo = min(min,0), hi = max(max,0)

By default the domain is the data extent [min, max], so the smallest-value bar has height 0. { zero: true } anchors the axis at zero (lo = least(min,0), hi = greatest(max,0)) so every bar has a real height and the labelled y axis is zero-based. The first constructor argument is the value column. Ticks are nice-linear.

Examples

Bare barY — domain is the data extent (the shortest bar collapses to zero height):

const bySpecies = db.table("penguins").groupby(["species"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
v.bar(bySpecies, {
  ...bandX("species", { paddingInner: 0.2 }).pick("x", "w"),
  ...barY("mass").pick("y", "h"),
  fill: ordinal("species"),
}).axis("x").axis("y");
return v;

Zero-based — { zero: true } gives every bar a real, comparable height:

const revenue = db.table("orders").rollup(
  db.table("products"), { left: "product_id", right: "id" },
  "category", { amount: sum({ sql: "qty * price" }) });
const v = canvas(640, 460);
v.bar(revenue, {
  ...bandX("category", { paddingInner: 0.15 }).pick("x", "w"),
  ...barY("amount", { zero: true }).pick("y", "h"),
  fill: ordinal("category"),
}).axis("x").axis("y");
return v;

barY gives a box you can label — a via borrows each solved bar box and a fitText sizes a label inside it ({ zero: true } guarantees a non-degenerate box):

const revenue = db.table("orders").rollup(
  db.table("products"), { left: "product_id", right: "id" },
  "category", { amount: sum({ sql: "qty * price" }) });
const v = canvas(640, 460);
const bars = v.bar(revenue, {
  ...bandX("category", { paddingInner: 0.15 }).pick("x", "w"),
  ...barY("amount", { zero: true }).pick("y", "h"),
  fill: ordinal("category"),
});
v.text(revenue, {
  ...bars.via(eq("category", "category")).pick({ x: "x", y: "y", w: "w", h: "h" }),
  ...fitText().pick("fontSize", "rotate"),
  text: field("category"),
  anchor: "middle", baseline: "middle", fill: "#fff",
});
return v;

Note. The label's text: field("category") uses field — the column expression from $lib/dvl/ir/expr (a raw data column). It is unrelated to the barY layout.

See also

  • bandX — the x-axis partner; category → x + width.
  • barX — the x-mirror; value → {x,w} horizontal bar from a baseline.
  • stackY — replace barY with stackY to fill a band with cumulative segments.
  • pitchY — fixed-pitch pixel bands instead of a value-scaled height.
