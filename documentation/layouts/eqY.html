eqY

eqY equal-splits a set of panels down the vertical range — the y-mirror of eqX. It distributes the distinct keys of an index column evenly and emits y (panel top) and h (panel height, the step) for each, bound with .pick("y","h"). It is the row-facet driver: the same rank→step shape as eqX, on the y axis. Outputs are normalized [0,1]; the frame affine turns them into pixels.

const species = db.table('penguins').groupby(['species'], {});  // one row per species (distinct)
const v = canvas(420, 620);
v.rect(species, { ...eqY('species').pick('y', 'h'), x: px(12), w: 396, fill: '#f5f6fa' })
 .dot('penguins',
   { x: linear('bill_length_mm'), y: linear('body_mass_g'), fill: '#4e79a7' },
   { on: 'species' });   // child.species = host.species
return v;

Stacked row facets — one panel per species, split evenly down y, each holding its own scatter.

Options & Outputs

eqY(key) takes a single input — the index/key column to split over. It fits a 1-row broadcast domain (countDistinct of the key) and emits equal shares: y = rank(key) / n, h = 1 / n. The fit is a bare count, so the domain is unshareable and unticked (no axis).

output space meaning
y relative coord y panel top = rank(key) / n
h relative len y panel height = 1 / n (the step)

eqY takes no options bag — spacing is exactly equal (use bandY if you want inner/outer padding).

See also

  • eqX — the x-mirror; distinct keys → equal horizontal split (column facets).
  • proportionalY — panels sized by a value instead of equally.
