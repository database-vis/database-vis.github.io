fitText

fitText is a refine layout: it consumes a mark's already-placed channels (post-placement pixels) rather than data. For every row it solves the largest fontSize, an optional rotate, and a line-wrap so the mark's own text fits inside its w×h box. Because it is a refine layout its inputs default to the same-named channel columns the mark already draws — so a bare fitText() sizes exactly the text this mark renders against the w/h box it sits in. You bind its outputs with .pick("fontSize", "rotate", "lines").

const revenue = db.table("orders").rollup(
  db.table("products"), { left: "product_id", right: "id" },
  "category", { amount: sum({ sql: "qty * price" }) });

const v = canvas(640, 460);
const bars = v.bar(revenue, {
  ...bandX("category", { paddingInner: 0.15 }).pick("x", "w"),
  ...barY("amount", { zero: true }).pick("y", "h"),
  fill: ordinal("category"),
});
// Borrow each bar's SOLVED box; anchor/baseline center the label inside it.
v.text(revenue, {
  ...bars.via("category").pick({ x: "x", y: "y", w: "w", h: "h" }),
  ...fitText().pick("fontSize", "rotate"),
  text: field("category"),
  anchor: "middle", baseline: "middle", fill: "#fff",
});
return v;

Each category label is sized to its own bar: tall bars take a big font, narrow tall bars rotate to -90°, short bars stay small and horizontal. This is the fit-text demo preset.

The refine model — where the box comes from

fitText reads text, w, and h from the mark it decorates. It never touches data or a domain (it trains nothing and is not tickable — fontSize/rotate/lines are terminal style outputs). The w/h must therefore already be pixel boxes. In practice you get them by .via(...).pick(...) from another mark's placed geometry — a bar's barY height, a treemap tile, a legend item — and fitText fits the string into that borrowed box. The label's text channel can be a literal (text: "Total") or a column (text: field("category")); fitText sizes whichever the mark draws.

Options

fitText(bag, options?). The single bag mixes input overrides and option keys freely because the two sets are disjoint — fitText({ rotate: true, maxFontSize: 14 }) reads naturally, and the two-arg fitText({ text: "label" }, { rotate: true }) form still works.

.pick(...) outputs

output space meaning
fontSize style(fontSize) solved point size (px), rounded to 0.5
rotate style(rotate) 0 or -90 — set when a rotated fit is larger
lines style(text) JSON array of wrapped lines (bind to a text mark that reads it)

Input overrides (default to the mark's same-named channel)

input default meaning
text the text channel the string to fit ({ text: "other_col" } to fit a different column)
w the w channel box width in px
h the h channel box height in px

Options bag

option type default meaning
rotate boolean true try a -90° fit; keep it when it yields a larger font. false pins horizontal
maxFontSize number Infinity ceiling. Uncapped means "as big as fits" — the box height still bounds it (fs·1.2 ≤ h), so the solve stays finite
minFontSize number 6 floor; a label that cannot fit is truncated with … rather than going below this
shared boolean false opt into ONE uniform size across all rows — the tightest box's solve caps the whole set
wrap boolean false break text on spaces into multiple lines to fit width
padding number 2 px inset subtracted from w/h before fitting

Defaults (a bare fitText()): per-box (shared: false), rotation tried (rotate: true), uncapped size bounded by the box (maxFontSize: Infinity).

Examples

Fit a label in a box

Bare fitText() — infer everything from the mark's channels, pick just the size:

const revenue = db.table("orders").rollup(
  db.table("products"), { left: "product_id", right: "id" },
  "category", { amount: sum({ sql: "qty * price" }) });
const v = canvas(640, 460);
const rects = v.rect(revenue, {
  ...treemap("amount").pick("x", "y", "w", "h"),
  fill: ordinal("category"), stroke: "#fff",
});
v.text(revenue, {
  ...rects.via("category").pick({ x: "x", y: "y", w: "w", h: "h" }),
  ...fitText().pick("fontSize"),
  text: field("category"), anchor: "middle", baseline: "middle", fill: "#333",
});
return v;

Each tile's label is sized to that tile. This is the treemap demo preset.

Rotate to fit a tall, narrow box

Rotation is on by default, so narrow tall boxes rotate automatically. Pick rotate too and apply it:

v.text(revenue, {
  ...bars.via("category", center).pick({ x: "cx", y: "cy", w: "w", h: "h" }),
  ...fitText().pick("fontSize", "rotate"),
  text: field("category"), anchor: "middle", baseline: "middle", fill: "#fff",
});

A long category over a slim tall bar is fitted at -90°; a short label in a wide box stays at 0°.

Pin horizontal

Turn rotation off when you never want sideways text:

v.text(revenue, {
  ...bars.via("category", center).pick({ x: "cx", y: "cy", w: "w", h: "h" }),
  ...fitText({ rotate: false }).pick("fontSize"),
  text: field("category"), anchor: "middle", baseline: "middle", fill: "#fff",
});

Cap the size (a finite ceiling)

Uncapped, a big box gives a big font. Cap it so labels never exceed 14px even when the box could hold more:

...fitText({ maxFontSize: 14 }).pick("fontSize"),

Raise the floor / avoid tiny truncated text

minFontSize (default 6) is the floor; below it, a too-long label is truncated with … instead of shrinking further:

...fitText({ minFontSize: 5, rotate: false }).pick("fontSize"),

Shared sizing across a set (uniform typography)

shared: true opts INTO one size for every row — the tightest box's solve caps the whole set, so all labels match. Useful across a facet or a group of tiles where varying font sizes would read as noise:

...fitText({ shared: true, maxFontSize: 14 }).pick("fontSize"),

Contrast the default (shared: false): each label solves against its own box, so sizes vary.

Wrap onto multiple lines

wrap: true breaks the text on spaces to fit width, and lines carries the wrapped array:

...fitText({ wrap: true }).pick("fontSize", "lines"),

Bind lines to a text mark that renders a multi-line label; fontSize is chosen so all lines fit the box height.

Override the fitted text column

By default fitText fits the same string the mark draws. Name text to fit a DIFFERENT column than the one rendered (e.g. size against a short code, draw the long name):

...fitText({ text: "code" }).pick("fontSize"),
text: field("name"),

Override the box columns

If the borrowed box columns are not named w/h, point fitText at them:

...someMark.via("id").pick({ x: "cx", y: "cy", w: "boxW", h: "boxH" }),
...fitText({ w: "boxW", h: "boxH" }).pick("fontSize"),

Tighten the padding

padding (default 2px) is inset from the box before fitting; set it to 0 for edge-to-edge text or higher to keep margin:

...fitText({ padding: 6 }).pick("fontSize"),
