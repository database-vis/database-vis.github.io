bandX

bandX is a layout operator: it turns a categorical column into a bar's horizontal placement. Like VL's band scale, but as a DVL multi-output layout — a bar needs TWO positional values on x (a left edge and a width), and a single channel binding carries one, so you bind both outputs with .pick("x","w"). Its trained fit IS the axis domain (one keyed (value, idx, n) table), so .axis("x") labels one tick per category, anchored at each band's center.

const bySpecies = db.table("penguins").groupby(["species"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
v.bar(bySpecies, {
  ...bandX("species").pick("x", "w"),   // the bar's x + width
  ...barY("mass").pick("y", "h"),       // the bar's top + height
  fill: ordinal("species"),
}).axis("x").axis("y");
return v;

A column chart: bandX spreads the species evenly across x, barY gives each bar its height.

Options

bandX emits normalized relative positions in [0,1] (the frame affine maps them into the x box). It has D3/VL band geometry: with n categories and inner/outer padding as fractions of the step,

step = 1 / (n − paddingInner + 2·paddingOuter)
left(i)  = (2·paddingOuter·align + i) / (n − paddingInner + 2·paddingOuter)
width    = (1 − paddingInner) · step

Outputs (bind with .pick(...)):

output space meaning
x relative coord (x) the band's LEFT edge
w relative len (x) the band width ((1−pi)·step)

Options bag (2nd arg, all static, baked at plan time):

option type default meaning
paddingInner number 0 gap between bands, fraction of step [0,1)
paddingOuter number 0 space before first / after last band, fraction of step
padding number — sets BOTH inner and outer (a fallback when either is unset)
align number 0.5 how outer padding is distributed [0,1]

The first constructor argument is the categorical column to place (bandX("species")). Ticks are ordinal — one per category, drawn from the domain rows.

Examples

A plain column chart (no padding — bars touch):

const bySpecies = db.table("penguins").groupby(["species"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
v.bar(bySpecies, {
  ...bandX("species").pick("x", "w"),
  ...barY("mass").pick("y", "h"),
  fill: ordinal("species"),
}).axis("x").axis("y");
return v;

Padded bars — a gap between bars via paddingInner:

const bySpecies = db.table("penguins").groupby(["species"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
v.bar(bySpecies, {
  ...bandX("species", { paddingInner: 0.2 }).pick("x", "w"),  // 20% of the step is gap
  ...barY("mass").pick("y", "h"),
  fill: ordinal("species"),
}).axis("x").axis("y");
return v;

Outer padding + alignment — inset the whole group and push it toward one side:

const bySpecies = db.table("penguins").groupby(["species"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
v.bar(bySpecies, {
  ...bandX("species", { paddingInner: 0.2, paddingOuter: 0.3, align: 0 }).pick("x", "w"),
  ...barY("mass").pick("y", "h"),
  fill: ordinal("species"),
}).axis("x").axis("y");
return v;

bandX supplies only x — pair it with stackY for stacked bars (bandX places, stackY fills):

const stacks = db.table("penguins").groupby(["species", "island"], { n: count() });
const v = canvas(640, 460);
v.rect(stacks, {
  ...bandX("species", { paddingInner: 0.2 }).pick("x", "w"),
  ...stackY({ weight: "n", group: "species", order: "island" }).pick("y", "h"),
  fill: ordinal("island"),
}).axis("x");
return v;

bandX on a joined/rolled-up measure — the x categories come from an FK traversal:

const revenue = db.table("orders").rollup(
  db.table("products"), { left: "product_id", right: "id" },
  "category", { amount: sum({ sql: "qty * price" }) });
const v = canvas(640, 460);
v.bar(revenue, {
  ...bandX("category", { paddingInner: 0.15 }).pick("x", "w"),
  ...barY("amount", { zero: true }).pick("y", "h"),
  fill: ordinal("category"),
}).axis("x").axis("y");
return v;

See also

  • barY — the y-axis partner; value → bar top + height.
  • bandY — the y-mirror; categorical → {y,h} (horizontal-bar category axis).
  • stackY — cumulative segments to fill each band (relative).
  • pitchY — fixed-pitch pixel bands (extent-independent).
