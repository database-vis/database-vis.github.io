pitchY

pitchY is a layout operator that stacks one fixed-height band per row down a column, ranked by an order column — the SQL-pushable legend-item stack. It emits y (the band TOP) and h (the band height), bound with .pick("y","h"). Unlike stackY, whose relative shares the frame STRETCHES to fill an extent, pitchY's outputs are PIXEL space and EXTENT-INDEPENDENT: the stack's total height is intrinsic (N·pitch), not the box's. Because it is a pure windowed derive (rank) with no box reads, it lowers to BOTH SQL and JS with no materialization — the point behind the autolegend's item stack.

const bySpecies = db.table("penguins").groupby(["species"], { n: count() });
const v = canvas(260, 220);
const bands = pitchY("species", { pitch: 30, pad: 16 });
v.rect(bySpecies, { ...bands.pick("y", "h"), x: 16, w: 22, fill: ordinal("species") });
v.text(bySpecies, { ...bands.pick("y", "h"), x: 48, text: field("species"),
                    fill: "#c5cbe0", anchor: "start" });
return v;

A hand-built legend column: one swatch rect + one label per species, ranked by name at a constant 30px pitch. Note the SAME pitchY instance is .pick-ed twice, so both marks share one ranking.

Options

pitchY measures pixel offsets DOWN from the stack's own origin:

y = pad + rank · (pitch + gap)   — the band TOP edge
h = pitch                        — the band height

rank is the 0-based dense rank of the order column, so distinct values get consecutive bands and rows that tie on the order column share a band. It is apply-only (no trained fit — nothing tickable). The first constructor argument is the order column (pitchY("species")).

Outputs (bind with .pick(...)):

output space meaning
y pixel coord (y) band TOP (pad + rank·step)
h pixel len (y) band height (pitch)

Options bag (2nd arg):

option type default meaning
pitch number 16 band height in pixels
gap number 0 extra gap between bands (step = pitch + gap)
pad number 0 offset of the first band from the origin

Examples

A swatch column — the canonical legend-item stack (share one instance across the two marks):

const bySpecies = db.table("penguins").groupby(["species"], { n: count() });
const v = canvas(260, 220);
const bands = pitchY("species", { pitch: 30, pad: 16 });
v.rect(bySpecies, { ...bands.pick("y", "h"), x: 16, w: 22, fill: ordinal("species") });
v.text(bySpecies, { ...bands.pick("y", "h"), x: 48, text: field("species"),
                    fill: "#c5cbe0", anchor: "start" });
return v;

Tighter pitch with a gap between swatches:

const bySpecies = db.table("penguins").groupby(["species"], { n: count() });
const v = canvas(200, 200);
const bands = pitchY("species", { pitch: 14, gap: 6, pad: 8 });
v.rect(bySpecies, { ...bands.pick("y", "h"), x: 12, w: 18, fill: ordinal("species") });
return v;

Any categorical ranks it — an island swatch column at the default 16px pitch:

const byIsland = db.table("penguins").groupby(["island"], { n: count() });
const v = canvas(200, 160);
const bands = pitchY("island");   // default pitch 16, gap 0, pad 0
v.rect(byIsland, { ...bands.pick("y", "h"), x: 12, w: 18, fill: ordinal("island") });
v.text(byIsland, { ...bands.pick("y", "h"), x: 40, text: field("island"), anchor: "start" });
return v;

Contrast with stackY. stackY fills a box (relative [0,1] shares that the frame stretches); pitchY is a fixed pixel pitch, so N items always occupy N·pitch pixels regardless of the box height — resize the canvas and a stackY column rescales while a pitchY column keeps its swatch size. Reach for pitchY for legends and item lists, stackY for stacked bars.
