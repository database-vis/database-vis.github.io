proportionalX

proportionalX sizes a strip of segments horizontally in proportion to a value column: each row's width is its share of the total, and each row's x is the running sum of the widths before it. It emits x (segment left) and w (segment width), both normalized to [0, 1] as cumulative shares of the sum (x = cumsum(preceding) / total, w = value / total). It is a one-axis partition — the horizontal cousin of a percent stack, and of proportionalY.

const revenue = db.table('orders').rollup(
  db.table('products'), { left: 'product_id', right: 'id' },
  'category', { amount: sum({ sql: 'qty * price' }) });
const v = canvas(720, 120);
v.rect(revenue, {
  ...proportionalX('amount').pick('x', 'w'),
  y: px(20), h: 80,
  fill: ordinal('category'),
  stroke: '#fff',
});
return v;

A single horizontal bar split into category segments whose widths are proportional to revenue — a marimekko/spine column. The measure is modelled inline: rollup joins orders→products and sums qty*price per category.

Options

proportionalX(value) takes a single numeric input. There is no trained domain — outputs are cumulative shares in [0, 1] (the same rationale as stackX), so nothing is tickable and a percent axis is not emitted. Negative values are clamped to 0; if the total is 0 the segments fall back to equal shares (i / n, 1 / n).

Input Type Meaning
value double the measure whose magnitude sets each segment's width

.pick() outputs:

Output Space Meaning
x relative coord x segment left = cumsum(preceding value) / total
w relative len x segment width = value / total

Bind by spreading into a channel object: ...proportionalX('amount').pick('x', 'w'). Row order along the strip follows the input relation's order. No options bag.

Examples

Categories by revenue, a spine chart (as above):

const revenue = db.table('orders').rollup(
  db.table('products'), { left: 'product_id', right: 'id' },
  'category', { amount: sum({ sql: 'qty * price' }) });
const v = canvas(720, 120);
v.rect(revenue, { ...proportionalX('amount').pick('x', 'w'), y: px(20), h: 80,
                  fill: ordinal('category'), stroke: '#fff' });
return v;

Penguin counts per species as a proportional strip — group first, then size by the count:

const bySpecies = db.table('penguins').groupby(['species'], { n: count() });
const v = canvas(640, 100);
v.rect(bySpecies, { ...proportionalX('n').pick('x', 'w'), y: px(20), h: 60,
                    fill: ordinal('species'), stroke: '#fff' });
return v;

Label each segment with a computed-center via — borrow the solved box and place text at its midpoint (f.x + f.w/2), fitted with fitText:

const revenue = db.table('orders').rollup(
  db.table('products'), { left: 'product_id', right: 'id' },
  'category', { amount: sum({ sql: 'qty * price' }) });
const v = canvas(760, 120);
const strip = v.rect(revenue, { ...proportionalX('amount').pick('x', 'w'), y: px(20), h: 80,
                                fill: ordinal('category'), stroke: '#fff' });
v.text(revenue, {
  ...strip.via(eq('category', 'category')).pick({ x: 'x', y: 'y', w: 'w', h: 'h' }),
  ...fitText(),
  text: field('category'), anchor: 'middle', baseline: 'middle', fill: '#fff',
});
return v;

Stacked proportional rows — pair it with a vertical layout for a two-axis mosaic. Here eqY splits the rows and each row's own proportionalX fills its width:

const revenue = db.table('orders').rollup(
  db.table('products'), { left: 'product_id', right: 'id' },
  'category', { amount: sum({ sql: 'qty * price' }) });
const v = canvas(720, 260);
const rows = db.table('penguins').groupby(['species'], {});
v.rect(rows, { ...eqY('species').pick('y', 'h'), x: px(10), w: 700, fill: '#f5f6fa' })
 .rect(revenue,
  { ...proportionalX('amount').pick('x', 'w'), y: px(4), h: 60, fill: ordinal('category'), stroke: '#fff' });
return v;
