barX

barX is a layout operator — the x-mirror of barY. It turns a numeric value into a horizontal bar growing from the baseline: it emits x (the bar's left edge) and w (its width out to the value), bound with .pick("x","w"). It trains a min/max fit like a linear scale and carries nice-linear ticks, so .axis("x") is labelled. barX is the value partner of bandY: bandY("cat").pick("y","h") gives {y,h}, barX("value").pick("x","w") gives {x,w}, and one rect/bar draws a horizontal bar.

const bySpecies = db.table("penguins").groupby(["species"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
v.bar(bySpecies, {
  ...bandY("species", { paddingInner: 0.2 }).pick("y", "h"),
  ...barX("mass", { zero: true }).pick("x", "w"),   // bar left + width from baseline
  fill: ordinal("species"),
}).axis("y").axis("x");
return v;

A horizontal bar chart of average body mass per species.

Options & Outputs

Outputs are normalized [0,1]: x is the value's position between the domain's lo/hi, and w is the same ratio treated as a length (baseline t=0).

output space meaning
x relative coord (x) the bar LEFT (value's position in the domain)
w relative len (x) the bar WIDTH out from the baseline

Options bag (2nd arg): zero: true anchors the domain at the baseline (lo = min(min,0), hi = max(max,0)) so every bar has a real width. Default domain is the data extent. Ticks are nice-linear.

See also

  • barY — the y-mirror; value → {y,h} vertical bar from the baseline.
  • bandY — the categorical partner; places each horizontal bar on the y axis.
