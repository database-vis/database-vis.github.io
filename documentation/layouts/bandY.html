bandY

bandY is a layout operator — the y-mirror of bandX. It turns a categorical column into a bar's vertical placement: it emits y (the band's top edge) and h (the band height), bound with .pick("y","h"). This is the category axis of a horizontal bar chart — pair it with barX, which supplies the bar's x + w. Its trained fit IS the axis domain, so .axis("y") labels one tick per category, centered on each band.

const bySpecies = db.table("penguins").groupby(["species"], { mass: avg("body_mass_g") });
const v = canvas(640, 460);
v.bar(bySpecies, {
  ...bandY("species", { paddingInner: 0.2 }).pick("y", "h"),   // the bar's y + height
  ...barX("mass").pick("x", "w"),                              // the bar's left + width
  fill: ordinal("species"),
}).axis("y").axis("x");
return v;

A horizontal bar chart: bandY spreads the species evenly down y, barX gives each bar its width.

Options & Outputs

Same D3/VL band geometry as bandX, on the y axis. Outputs are normalized [0,1]; the frame affine maps them into the y box.

output space meaning
y relative coord (y) the band's TOP edge
h relative len (y) the band height ((1−pi)·step)

Options bag (2nd arg): paddingInner, paddingOuter, padding, align — identical semantics to bandX. Ticks are ordinal, one per category.

See also

  • bandX — the x-mirror; categorical → {x,w} (vertical-bar category axis).
  • barX — the value partner; horizontal bar width from a baseline.
